## Getting Started  

ਇਸ ਭਾਗ ਵਿੱਚ ਕਈ ਪਾਠ ਹਨ:

- **1 ਤੁਹਾਡਾ ਪਹਿਲਾ ਸਰਵਰ**, ਇਸ ਪਹਿਲੇ ਪਾਠ ਵਿੱਚ, ਤੁਸੀਂ ਸਿੱਖੋਗੇ ਕਿ ਆਪਣਾ ਪਹਿਲਾ ਸਰਵਰ ਕਿਵੇਂ ਬਣਾਇਆ ਜਾਂਦਾ ਹੈ ਅਤੇ ਇਸਨੂੰ ਇੰਸਪੈਕਟਰ ਟੂਲ ਨਾਲ ਕਿਵੇਂ ਜਾਂਚਿਆ ਜਾਂਦਾ ਹੈ, ਜੋ ਕਿ ਤੁਹਾਡੇ ਸਰਵਰ ਦੀ ਟੈਸਟਿੰਗ ਅਤੇ ਡੀਬੱਗਿੰਗ ਲਈ ਇੱਕ ਕੀਮਤੀ ਤਰੀਕਾ ਹੈ, [ਪਾਠ ਵੱਲ](01-first-server/README.md)

- **2 ਕਲਾਇੰਟ**, ਇਸ ਪਾਠ ਵਿੱਚ, ਤੁਸੀਂ ਸਿੱਖੋਗੇ ਕਿ ਕਿਵੇਂ ਇੱਕ ਕਲਾਇੰਟ ਲਿਖਿਆ ਜਾਂਦਾ ਹੈ ਜੋ ਤੁਹਾਡੇ ਸਰਵਰ ਨਾਲ ਜੁੜ ਸਕਦਾ ਹੈ, [ਪਾਠ ਵੱਲ](02-client/README.md)

- **3 LLM ਨਾਲ ਕਲਾਇੰਟ**, ਕਲਾਇੰਟ ਲਿਖਣ ਦਾ ਇੱਕ ਹੋਰ ਵਧੀਆ ਤਰੀਕਾ ਹੈ ਕਿ ਇਸ ਵਿੱਚ LLM ਸ਼ਾਮਲ ਕੀਤਾ ਜਾਵੇ ਤਾਂ ਜੋ ਇਹ ਤੁਹਾਡੇ ਸਰਵਰ ਨਾਲ "ਮੁਲਾਕਾਤ" ਕਰ ਸਕੇ ਕਿ ਕੀ ਕਰਨਾ ਹੈ, [ਪਾਠ ਵੱਲ](03-llm-client/README.md)

- **4 Visual Studio Code ਵਿੱਚ GitHub Copilot Agent ਮੋਡ ਵਿੱਚ ਸਰਵਰ ਦੀ ਵਰਤੋਂ**। ਇੱਥੇ ਅਸੀਂ Visual Studio Code ਦੇ ਅੰਦਰੋਂ ਆਪਣਾ MCP ਸਰਵਰ ਚਲਾਉਣ ਬਾਰੇ ਵੇਖ ਰਹੇ ਹਾਂ, [ਪਾਠ ਵੱਲ](04-vscode/README.md)

- **5 SSE (Server Sent Events) ਤੋਂ ਵਰਤੋਂ** SSE ਇੱਕ ਮਿਆਰੀ ਤਰੀਕਾ ਹੈ ਸਰਵਰ ਤੋਂ ਕਲਾਇੰਟ ਤੱਕ ਸਟ੍ਰੀਮਿੰਗ ਲਈ, ਜੋ ਸਰਵਰਾਂ ਨੂੰ HTTP ਰਾਹੀਂ ਕਲਾਇੰਟਾਂ ਨੂੰ ਰੀਅਲ-ਟਾਈਮ ਅੱਪਡੇਟ ਭੇਜਣ ਦੀ ਆਗਿਆ ਦਿੰਦਾ ਹੈ [ਪਾਠ ਵੱਲ](05-sse-server/README.md)

- **6 MCP ਨਾਲ HTTP ਸਟ੍ਰੀਮਿੰਗ (Streamable HTTP)**। ਆਧੁਨਿਕ HTTP ਸਟ੍ਰੀਮਿੰਗ, ਪ੍ਰਗਤੀ ਸੂਚਨਾਵਾਂ ਅਤੇ ਕਿਵੇਂ ਸਕੇਲ ਕਰਨ ਯੋਗ, ਰੀਅਲ-ਟਾਈਮ MCP ਸਰਵਰ ਅਤੇ ਕਲਾਇੰਟ ਬਣਾਏ ਜਾ ਸਕਦੇ ਹਨ Streamable HTTP ਦੀ ਵਰਤੋਂ ਨਾਲ, ਬਾਰੇ ਸਿੱਖੋ। [ਪਾਠ ਵੱਲ](06-http-streaming/README.md)

- **7 VSCode ਲਈ AI Toolkit ਦੀ ਵਰਤੋਂ** ਆਪਣੇ MCP ਕਲਾਇੰਟ ਅਤੇ ਸਰਵਰਾਂ ਦੀ ਟੈਸਟਿੰਗ ਅਤੇ ਵਰਤੋਂ ਲਈ [ਪਾਠ ਵੱਲ](07-aitk/README.md)

- **8 ਟੈਸਟਿੰਗ**। ਇੱਥੇ ਅਸੀਂ ਖਾਸ ਤੌਰ 'ਤੇ ਦੇਖਾਂਗੇ ਕਿ ਕਿਵੇਂ ਅਸੀਂ ਆਪਣੇ ਸਰਵਰ ਅਤੇ ਕਲਾਇੰਟ ਨੂੰ ਵੱਖ-ਵੱਖ ਤਰੀਕਿਆਂ ਨਾਲ ਟੈਸਟ ਕਰ ਸਕਦੇ ਹਾਂ, [ਪਾਠ ਵੱਲ](08-testing/README.md)

- **9 ਡਿਪਲੋਇਮੈਂਟ**। ਇਹ ਅਧਿਆਇ ਤੁਹਾਡੇ MCP ਹੱਲਾਂ ਨੂੰ ਡਿਪਲੋਇ ਕਰਨ ਦੇ ਵੱਖ-ਵੱਖ ਤਰੀਕਿਆਂ ਬਾਰੇ ਜਾਣੂ ਕਰਵਾਏਗਾ, [ਪਾਠ ਵੱਲ](09-deployment/README.md)


Model Context Protocol (MCP) ਇੱਕ ਖੁੱਲ੍ਹਾ ਪ੍ਰੋਟੋਕੋਲ ਹੈ ਜੋ ਇਹ ਨਿਰਧਾਰਤ ਕਰਦਾ ਹੈ ਕਿ ਐਪਲੀਕੇਸ਼ਨ LLMs ਨੂੰ ਸੰਦਰਭ ਕਿਵੇਂ ਪ੍ਰਦਾਨ ਕਰਦੇ ਹਨ। MCP ਨੂੰ AI ਐਪਲੀਕੇਸ਼ਨਾਂ ਲਈ USB-C ਪੋਰਟ ਵਾਂਗ ਸੋਚੋ - ਇਹ AI ਮਾਡਲਾਂ ਨੂੰ ਵੱਖ-ਵੱਖ ਡਾਟਾ ਸਰੋਤਾਂ ਅਤੇ ਟੂਲਾਂ ਨਾਲ ਜੁੜਨ ਦਾ ਇੱਕ ਮਿਆਰੀ ਤਰੀਕਾ ਦਿੰਦਾ ਹੈ।

## Learning Objectives

ਇਸ ਪਾਠ ਦੇ ਅੰਤ ਤੱਕ, ਤੁਸੀਂ ਸਮਰੱਥ ਹੋਵੋਗੇ:

- C#, Java, Python, TypeScript, ਅਤੇ JavaScript ਵਿੱਚ MCP ਲਈ ਵਿਕਾਸ ਵਾਤਾਵਰਣ ਸੈੱਟਅਪ ਕਰਨਾ
- ਕਸਟਮ ਫੀਚਰਾਂ (ਸੰਸਾਧਨ, ਪ੍ਰਾਂਪਟ, ਅਤੇ ਟੂਲ) ਨਾਲ ਬੁਨਿਆਦੀ MCP ਸਰਵਰ ਬਣਾਉਣਾ ਅਤੇ ਡਿਪਲੋਇ ਕਰਨਾ
- MCP ਸਰਵਰਾਂ ਨਾਲ ਜੁੜਨ ਵਾਲੀਆਂ ਹੋਸਟ ਐਪਲੀਕੇਸ਼ਨਾਂ ਬਣਾਉਣਾ
- MCP ਇੰਪਲੀਮੈਂਟੇਸ਼ਨਾਂ ਦੀ ਟੈਸਟਿੰਗ ਅਤੇ ਡੀਬੱਗਿੰਗ ਕਰਨਾ
- ਆਮ ਸੈੱਟਅਪ ਚੁਣੌਤੀਆਂ ਅਤੇ ਉਨ੍ਹਾਂ ਦੇ ਹੱਲ ਸਮਝਣਾ
- ਆਪਣੇ MCP ਇੰਪਲੀਮੈਂਟੇਸ਼ਨਾਂ ਨੂੰ ਪ੍ਰਸਿੱਧ LLM ਸੇਵਾਵਾਂ ਨਾਲ ਜੋੜਨਾ

## Setting Up Your MCP Environment

MCP ਨਾਲ ਕੰਮ ਸ਼ੁਰੂ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ, ਇਹ ਜਰੂਰੀ ਹੈ ਕਿ ਤੁਸੀਂ ਆਪਣਾ ਵਿਕਾਸ ਵਾਤਾਵਰਣ ਤਿਆਰ ਕਰੋ ਅਤੇ ਮੂਲ ਵਰਕਫਲੋ ਨੂੰ ਸਮਝੋ। ਇਹ ਭਾਗ ਤੁਹਾਨੂੰ ਸ਼ੁਰੂਆਤੀ ਸੈੱਟਅਪ ਕਦਮਾਂ ਵਿੱਚ ਮਦਦ ਕਰੇਗਾ ਤਾਂ ਜੋ MCP ਨਾਲ ਸਹੀ ਤਰੀਕੇ ਨਾਲ ਸ਼ੁਰੂਆਤ ਹੋ ਸਕੇ।

### Prerequisites

MCP ਵਿਕਾਸ ਵਿੱਚ ਡੁੱਬਕੀਆਂ ਮਾਰਨ ਤੋਂ ਪਹਿਲਾਂ, ਇਹ ਯਕੀਨੀ ਬਣਾਓ ਕਿ ਤੁਹਾਡੇ ਕੋਲ ਹੈ:

- **ਵਿਕਾਸ ਵਾਤਾਵਰਣ**: ਆਪਣੀ ਚੁਣੀ ਹੋਈ ਭਾਸ਼ਾ ਲਈ (C#, Java, Python, TypeScript, ਜਾਂ JavaScript)
- **IDE/Editor**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm, ਜਾਂ ਕੋਈ ਵੀ ਆਧੁਨਿਕ ਕੋਡ ਐਡੀਟਰ
- **ਪੈਕੇਜ ਮੈਨੇਜਰ**: NuGet, Maven/Gradle, pip, ਜਾਂ npm/yarn
- **API Keys**: ਕਿਸੇ ਵੀ AI ਸੇਵਾਵਾਂ ਲਈ ਜੋ ਤੁਸੀਂ ਆਪਣੇ ਹੋਸਟ ਐਪਲੀਕੇਸ਼ਨਾਂ ਵਿੱਚ ਵਰਤਣਾ ਚਾਹੁੰਦੇ ਹੋ


### Official SDKs

ਅਗਲੇ ਅਧਿਆਇਆਂ ਵਿੱਚ ਤੁਸੀਂ Python, TypeScript, Java ਅਤੇ .NET ਦੀ ਵਰਤੋਂ ਨਾਲ ਬਣਾਈਆਂ ਗਈਆਂ ਹੱਲਾਂ ਵੇਖੋਗੇ। ਇੱਥੇ ਸਾਰੇ ਅਧਿਕਾਰਿਕ ਤੌਰ 'ਤੇ ਸਮਰਥਿਤ SDKs ਹਨ।

MCP ਕਈ ਭਾਸ਼ਾਵਾਂ ਲਈ ਅਧਿਕਾਰਿਕ SDKs ਪ੍ਰਦਾਨ ਕਰਦਾ ਹੈ:
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - Microsoft ਨਾਲ ਸਹਿਯੋਗ ਵਿੱਚ ਸੰਭਾਲਿਆ ਗਿਆ
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - Spring AI ਨਾਲ ਸਹਿਯੋਗ ਵਿੱਚ ਸੰਭਾਲਿਆ ਗਿਆ
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - ਅਧਿਕਾਰਿਕ TypeScript ਇੰਪਲੀਮੈਂਟੇਸ਼ਨ
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - ਅਧਿਕਾਰਿਕ Python ਇੰਪਲੀਮੈਂਟੇਸ਼ਨ
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - ਅਧਿਕਾਰਿਕ Kotlin ਇੰਪਲੀਮੈਂਟੇਸ਼ਨ
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - Loopwork AI ਨਾਲ ਸਹਿਯੋਗ ਵਿੱਚ ਸੰਭਾਲਿਆ ਗਿਆ
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - ਅਧਿਕਾਰਿਕ Rust ਇੰਪਲੀਮੈਂਟੇਸ਼ਨ

## Key Takeaways

- MCP ਵਿਕਾਸ ਵਾਤਾਵਰਣ ਸੈੱਟਅਪ ਕਰਨਾ ਭਾਸ਼ਾ-ਵਿਸ਼ੇਸ਼ SDKs ਨਾਲ ਸੌਖਾ ਹੈ
- MCP ਸਰਵਰ ਬਣਾਉਣ ਵਿੱਚ ਸਾਫ਼ ਸਕੀਮਾਂ ਨਾਲ ਟੂਲ ਬਣਾਉਣਾ ਅਤੇ ਰਜਿਸਟਰ ਕਰਨਾ ਸ਼ਾਮਲ ਹੈ
- MCP ਕਲਾਇੰਟ ਸਰਵਰਾਂ ਅਤੇ ਮਾਡਲਾਂ ਨਾਲ ਜੁੜਦੇ ਹਨ ਤਾਂ ਜੋ ਵਧੇਰੇ ਸਮਰੱਥਾਵਾਂ ਦਾ ਲਾਭ ਲਿਆ ਜਾ ਸਕੇ
- ਟੈਸਟਿੰਗ ਅਤੇ ਡੀਬੱਗਿੰਗ MCP ਇੰਪਲੀਮੈਂਟੇਸ਼ਨਾਂ ਲਈ ਜਰੂਰੀ ਹਨ
- ਡਿਪਲੋਇਮੈਂਟ ਦੇ ਵਿਕਲਪ ਸਥਾਨਕ ਵਿਕਾਸ ਤੋਂ ਲੈ ਕੇ ਕਲਾਉਡ-ਆਧਾਰਿਤ ਹੱਲਾਂ ਤੱਕ ਹਨ
