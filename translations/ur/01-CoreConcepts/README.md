# 📖 MCP Core Concepts: ماڈل کانٹیکسٹ پروٹوکول برائے AI انٹیگریشن میں مہارت حاصل کرنا

Model Context Protocol (MCP) ایک طاقتور، معیاری فریم ورک ہے جو بڑے زبان کے ماڈلز (LLMs) اور بیرونی ٹولز، ایپلیکیشنز، اور ڈیٹا ذرائع کے درمیان مواصلات کو بہتر بناتا ہے۔ یہ SEO کے لحاظ سے بہتر بنایا گیا گائیڈ آپ کو MCP کے بنیادی تصورات سے روشناس کرائے گا، تاکہ آپ اس کی کلائنٹ-سرور آرکیٹیکچر، اہم اجزاء، مواصلاتی طریقہ کار، اور بہترین نفاذ کے طریقے سمجھ سکیں۔

## جائزہ

یہ سبق Model Context Protocol (MCP) کے بنیادی آرکیٹیکچر اور اجزاء کا جائزہ لیتا ہے جو MCP ماحولیاتی نظام کو تشکیل دیتے ہیں۔ آپ کلائنٹ-سرور آرکیٹیکچر، اہم اجزاء، اور مواصلاتی طریقہ کار کے بارے میں جانیں گے جو MCP تعاملات کو ممکن بناتے ہیں۔

## 👩‍🎓 کلیدی سیکھنے کے مقاصد

اس سبق کے آخر تک آپ:

- MCP کلائنٹ-سرور آرکیٹیکچر کو سمجھ جائیں گے۔
- Hosts, Clients, اور Servers کے کردار اور ذمہ داریاں پہچانیں گے۔
- MCP کو ایک لچکدار انٹیگریشن پرت بنانے والی بنیادی خصوصیات کا تجزیہ کریں گے۔
- MCP ماحولیاتی نظام میں معلومات کے بہاؤ کو سمجھیں گے۔
- .NET، Java، Python، اور JavaScript میں کوڈ کی مثالوں کے ذریعے عملی بصیرت حاصل کریں گے۔

## 🔎 MCP آرکیٹیکچر: ایک گہرا جائزہ

MCP ماحولیاتی نظام کلائنٹ-سرور ماڈل پر مبنی ہے۔ یہ ماڈیولر ڈھانچہ AI ایپلیکیشنز کو ٹولز، ڈیٹا بیسز، APIs، اور سیاق و سباق کے وسائل کے ساتھ مؤثر انداز میں تعامل کی اجازت دیتا ہے۔ آئیے اس آرکیٹیکچر کو اس کے بنیادی اجزاء میں تقسیم کرتے ہیں۔

### 1. Hosts

Model Context Protocol (MCP) میں، Hosts ایک اہم کردار ادا کرتے ہیں کیونکہ یہ وہ بنیادی انٹرفیس ہیں جن کے ذریعے صارفین پروٹوکول کے ساتھ تعامل کرتے ہیں۔ Hosts وہ ایپلیکیشنز یا ماحولیات ہیں جو MCP سرورز کے ساتھ کنکشن شروع کرتے ہیں تاکہ ڈیٹا، ٹولز، اور پرامپٹس تک رسائی حاصل کی جا سکے۔ Hosts کی مثالوں میں Visual Studio Code جیسے انٹیگریٹڈ ڈیولپمنٹ ماحول (IDEs)، Claude Desktop جیسے AI ٹولز، یا مخصوص کاموں کے لیے تیار کیے گئے کسٹم ایجنٹس شامل ہیں۔

**Hosts** وہ LLM ایپلیکیشنز ہیں جو کنکشن شروع کرتے ہیں۔ یہ:

- AI ماڈلز کے ساتھ بات چیت یا عمل درآمد کرتے ہیں تاکہ جوابات پیدا کریں۔
- MCP سرورز کے ساتھ کنکشن شروع کرتے ہیں۔
- گفتگو کے بہاؤ اور صارف کے انٹرفیس کا انتظام کرتے ہیں۔
- اجازت اور سیکیورٹی پابندیوں کو کنٹرول کرتے ہیں۔
- ڈیٹا شیئرنگ اور ٹول کے عمل کے لیے صارف کی رضامندی سنبھالتے ہیں۔

### 2. Clients

Clients وہ اہم اجزاء ہیں جو Hosts اور MCP سرورز کے درمیان تعامل کو آسان بناتے ہیں۔ Clients ایک درمیانی حیثیت رکھتے ہیں، جو Hosts کو MCP سرورز کی فراہم کردہ فعالیت تک رسائی اور استعمال کی اجازت دیتے ہیں۔ یہ MCP آرکیٹیکچر کے اندر ہموار مواصلات اور مؤثر ڈیٹا تبادلے کو یقینی بنانے میں اہم کردار ادا کرتے ہیں۔

**Clients** host ایپلیکیشن کے اندر کنیکٹرز ہیں۔ یہ:

- سرورز کو پرامپٹس/ہدایات کے ساتھ درخواستیں بھیجتے ہیں۔
- سرورز کے ساتھ صلاحیتوں پر مذاکرات کرتے ہیں۔
- ماڈلز سے ٹول کے عمل درآمد کی درخواستوں کا انتظام کرتے ہیں۔
- صارفین کو جوابات کو پروسیس اور دکھاتے ہیں۔

### 3. Servers

Servers MCP clients سے درخواستیں وصول کرنے اور مناسب جوابات فراہم کرنے کے ذمہ دار ہوتے ہیں۔ یہ مختلف آپریشنز جیسے ڈیٹا بازیافت، ٹول کے عمل درآمد، اور پرامپٹ جنریشن کو سنبھالتے ہیں۔ Servers اس بات کو یقینی بناتے ہیں کہ clients اور Hosts کے درمیان مواصلات مؤثر اور قابل اعتماد ہوں، اور تعامل کے عمل کی سالمیت برقرار رکھیں۔

**Servers** وہ خدمات ہیں جو سیاق و سباق اور صلاحیتیں فراہم کرتی ہیں۔ یہ:

- دستیاب خصوصیات (وسائل، پرامپٹس، ٹولز) کو رجسٹر کرتے ہیں۔
- کلائنٹ سے ٹول کالز وصول کرتے اور عمل کرتے ہیں۔
- ماڈل کے جوابات کو بہتر بنانے کے لیے سیاق و سباق کی معلومات فراہم کرتے ہیں۔
- نتائج کو کلائنٹ کو واپس بھیجتے ہیں۔
- ضرورت پڑنے پر تعاملات کے دوران حالت کو برقرار رکھتے ہیں۔

Servers کو کوئی بھی تیار کر سکتا ہے تاکہ ماڈل کی صلاحیتوں کو خصوصی فعالیت کے ساتھ بڑھایا جا سکے۔

### 4. Server Features

Model Context Protocol (MCP) میں Servers بنیادی تعمیراتی بلاکس فراہم کرتے ہیں جو clients، hosts، اور زبان کے ماڈلز کے درمیان بھرپور تعاملات کو ممکن بناتے ہیں۔ یہ خصوصیات MCP کی صلاحیتوں کو منظم سیاق و سباق، ٹولز، اور پرامپٹس پیش کر کے بڑھانے کے لیے ڈیزائن کی گئی ہیں۔

MCP سرورز درج ذیل خصوصیات میں سے کوئی بھی پیش کر سکتے ہیں:

#### 📑 Resources

Model Context Protocol (MCP) میں Resources مختلف قسم کے سیاق و سباق اور ڈیٹا پر مشتمل ہوتے ہیں جنہیں صارفین یا AI ماڈلز استعمال کر سکتے ہیں۔ ان میں شامل ہیں:

- **سیاق و سباق کا ڈیٹا**: معلومات اور سیاق و سباق جو صارفین یا AI ماڈلز فیصلہ سازی اور کاموں کی انجام دہی کے لیے استعمال کر سکتے ہیں۔
- **علمی بنیادیں اور دستاویزی ذخائر**: ساختہ اور غیر ساختہ ڈیٹا کے مجموعے، جیسے مضامین، دستی کتابیں، اور تحقیقی مقالے، جو قیمتی بصیرت اور معلومات فراہم کرتے ہیں۔
- **مقامی فائلیں اور ڈیٹا بیسز**: ڈیٹا جو مقامی طور پر آلات پر یا ڈیٹا بیسز میں محفوظ ہوتا ہے، جس تک پروسیسنگ اور تجزیہ کے لیے رسائی ممکن ہے۔
- **APIs اور ویب سروسز**: بیرونی انٹرفیس اور خدمات جو اضافی ڈیٹا اور فعالیت فراہم کرتی ہیں، مختلف آن لائن وسائل اور ٹولز کے ساتھ انضمام کو ممکن بناتی ہیں۔

Resource کی ایک مثال ڈیٹا بیس اسکیمہ یا فائل ہو سکتی ہے جس تک اس طرح رسائی حاصل کی جا سکتی ہے:

```text
file://log.txt
database://schema
```

### 🤖 Prompts

Model Context Protocol (MCP) میں Prompts مختلف پیشگی متعین ٹیمپلیٹس اور تعامل کے نمونے شامل ہوتے ہیں جو صارف کے ورک فلو کو آسان بنانے اور مواصلات کو بہتر بنانے کے لیے بنائے گئے ہیں۔ ان میں شامل ہیں:

- **ٹیملیٹڈ پیغامات اور ورک فلو**: پیش ساختہ پیغامات اور عمل جو صارفین کو مخصوص کاموں اور تعاملات کے ذریعے رہنمائی کرتے ہیں۔
- **پیشگی متعین تعامل کے نمونے**: معیاری کارروائیوں اور جوابات کے تسلسل جو مستقل اور مؤثر مواصلات کو فروغ دیتے ہیں۔
- **خصوصی بات چیت کے ٹیمپلیٹس**: مخصوص قسم کی بات چیت کے لیے حسب ضرورت ٹیمپلیٹس، جو متعلقہ اور سیاق و سباق کے لحاظ سے مناسب تعاملات کو یقینی بناتے ہیں۔

ایک پرامپٹ ٹیمپلیٹ اس طرح دکھائی دے سکتا ہے:

```markdown
Generate a product slogan based on the following {{product}} with the following {{keywords}}
```

#### ⛏️ Tools

Model Context Protocol (MCP) میں Tools وہ فنکشنز ہیں جنہیں AI ماڈل مخصوص کام انجام دینے کے لیے چلا سکتا ہے۔ یہ ٹولز AI ماڈل کی صلاحیتوں کو منظم اور قابل اعتماد آپریشنز فراہم کر کے بڑھانے کے لیے ڈیزائن کیے گئے ہیں۔ اہم پہلوؤں میں شامل ہیں:

- **AI ماڈل کے لیے قابل عمل فنکشنز**: ٹولز ایسے فنکشنز ہیں جنہیں AI ماڈل مختلف کاموں کے لیے بلا سکتا ہے۔
- **منفرد نام اور وضاحت**: ہر ٹول کا ایک مخصوص نام اور تفصیلی وضاحت ہوتی ہے جو اس کے مقصد اور فعالیت کی وضاحت کرتی ہے۔
- **پیرا میٹرز اور آؤٹ پٹ**: ٹولز مخصوص پیرا میٹرز قبول کرتے ہیں اور منظم نتائج واپس کرتے ہیں، جو مستقل اور متوقع نتائج کو یقینی بناتے ہیں۔
- **مخصوص فنکشنز**: ٹولز مخصوص کام انجام دیتے ہیں جیسے ویب سرچز، حساب کتاب، اور ڈیٹا بیس کی کوئریز۔

ایک ٹول کی مثال اس طرح ہو سکتی ہے:

```typescript
server.tool(
  "GetProducts",
  {
    pageSize: z.string().optional(),
    pageCount: z.string().optional()
  }, () => {
    // return results from API
  }
)
```

## Client Features

Model Context Protocol (MCP) میں، clients سرورز کو کئی کلیدی خصوصیات فراہم کرتے ہیں جو پروٹوکول کے اندر مجموعی فعالیت اور تعامل کو بڑھاتی ہیں۔ ان خصوصیات میں سے ایک نمایاں خصوصیت Sampling ہے۔

### 👉 Sampling

- **سرور کی جانب سے شروع کردہ ایجنٹک رویے**: clients سرورز کو خود مختار طور پر مخصوص کارروائیاں یا رویے شروع کرنے کی اجازت دیتے ہیں، جو نظام کی متحرک صلاحیتوں کو بڑھاتے ہیں۔
- **Recursive LLM Interactions**: یہ خصوصیت بڑے زبان کے ماڈلز (LLMs) کے ساتھ تکراری تعاملات کی اجازت دیتی ہے، جو کاموں کی مزید پیچیدہ اور تدریجی پروسیسنگ ممکن بناتی ہے۔
- **اضافی ماڈل کمپلیشنز کی درخواست**: سرورز ماڈل سے اضافی جوابات طلب کر سکتے ہیں، تاکہ جوابات مکمل اور سیاق و سباق کے لحاظ سے متعلقہ ہوں۔

## MCP میں معلومات کا بہاؤ

Model Context Protocol (MCP) میزبانوں، کلائنٹس، سرورز، اور ماڈلز کے درمیان معلومات کے منظم بہاؤ کی وضاحت کرتا ہے۔ اس بہاؤ کو سمجھنے سے یہ واضح ہوتا ہے کہ صارف کی درخواستیں کیسے پروسیس ہوتی ہیں اور بیرونی ٹولز اور ڈیٹا کس طرح ماڈل کے جوابات میں شامل کیے جاتے ہیں۔

- **Host کنکشن شروع کرتا ہے**  
  میزبان ایپلیکیشن (جیسے IDE یا چیٹ انٹرفیس) عام طور پر STDIO، WebSocket، یا کسی دوسرے سپورٹڈ ٹرانسپورٹ کے ذریعے MCP سرور سے کنکشن قائم کرتا ہے۔

- **صلاحیتوں پر مذاکرات**  
  کلائنٹ (جو میزبان میں شامل ہوتا ہے) اور سرور اپنی سپورٹ کردہ خصوصیات، ٹولز، وسائل، اور پروٹوکول ورژنز کے بارے میں معلومات کا تبادلہ کرتے ہیں۔ یہ اس بات کو یقینی بناتا ہے کہ دونوں طرفیں سیشن کے لیے دستیاب صلاحیتوں کو سمجھتی ہیں۔

- **صارف کی درخواست**  
  صارف میزبان کے ساتھ بات چیت کرتا ہے (مثلاً کوئی پرامپٹ یا کمانڈ داخل کرتا ہے)۔ میزبان یہ ان پٹ جمع کرتا ہے اور اسے پروسیسنگ کے لیے کلائنٹ کو بھیجتا ہے۔

- **وسائل یا ٹول کا استعمال**  
  - کلائنٹ سرور سے اضافی سیاق و سباق یا وسائل طلب کر سکتا ہے (جیسے فائلیں، ڈیٹا بیس اندراجات، یا علمی مضامین) تاکہ ماڈل کی سمجھ کو بڑھایا جا سکے۔  
  - اگر ماڈل طے کرتا ہے کہ ٹول کی ضرورت ہے (مثلاً ڈیٹا حاصل کرنے، حساب کتاب کرنے، یا API کال کرنے کے لیے)، تو کلائنٹ ٹول کال کی درخواست سرور کو بھیجتا ہے، جس میں ٹول کا نام اور پیرا میٹرز شامل ہوتے ہیں۔

- **سرور کا عمل درآمد**  
  سرور وسائل یا ٹول کی درخواست وصول کرتا ہے، ضروری آپریشنز انجام دیتا ہے (جیسے فنکشن چلانا، ڈیٹا بیس سے سوال کرنا، یا فائل بازیافت کرنا)، اور نتائج کو منظم شکل میں کلائنٹ کو واپس بھیجتا ہے۔

- **جواب کی تیاری**  
  کلائنٹ سرور کے جوابات (وسائل کا ڈیٹا، ٹول آؤٹ پٹ، وغیرہ) کو جاری ماڈل تعامل میں شامل کرتا ہے۔ ماڈل اس معلومات کا استعمال کرتے ہوئے جامع اور سیاق و سباق کے لحاظ سے متعلقہ جواب تیار کرتا ہے۔

- **نتائج کی پیشکش**  
  میزبان کلائنٹ سے حتمی آؤٹ پٹ وصول کرتا ہے اور اسے صارف کو پیش کرتا ہے، جس میں اکثر ماڈل کی تیار کردہ تحریر اور ٹول کے عمل یا وسائل کی تلاش کے نتائج شامل ہوتے ہیں۔

یہ بہاؤ MCP کو جدید، انٹرایکٹو، اور سیاق و سباق سے آگاہ AI ایپلیکیشنز کی حمایت کرنے کے قابل بناتا ہے، جو ماڈلز کو بیرونی ٹولز اور ڈیٹا ذرائع سے بلا رکاوٹ جوڑتا ہے۔

## پروٹوکول کی تفصیلات

MCP (Model Context Protocol) [JSON-RPC 2.0](https://www.jsonrpc.org/) کے اوپر بنایا گیا ہے، جو میزبانوں، کلائنٹس، اور سرورز کے درمیان مواصلات کے لیے ایک معیاری، زبان سے آزاد پیغام کا فارمیٹ فراہم کرتا ہے۔ یہ بنیاد مختلف پلیٹ فارمز اور پروگرامنگ زبانوں میں قابل اعتماد، منظم، اور قابل توسیع تعاملات کو ممکن بناتی ہے۔

### کلیدی پروٹوکول خصوصیات

MCP JSON-RPC 2.0 کو اضافی روایات کے ساتھ بڑھاتا ہے جو ٹول کال، وسائل تک رسائی، اور پرامپٹ مینجمنٹ کے لیے ہیں۔ یہ متعدد ٹرانسپورٹ پرتوں (STDIO، WebSocket، SSE) کو سپورٹ کرتا ہے اور اجزاء کے درمیان محفوظ، قابل توسیع، اور زبان سے آزاد مواصلات کو فعال بناتا ہے۔

#### 🧢 بنیادی پروٹوکول

- **JSON-RPC میسج فارمیٹ**: تمام درخواستیں اور جوابات JSON-RPC 2.0 وضاحت استعمال کرتے ہیں، جو طریقہ کار کی کالز، پیرا میٹرز، نتائج، اور ایرر ہینڈلنگ کے لیے مستقل ساخت کو یقینی بناتا ہے۔
- **حالت دار کنکشنز**: MCP سیشنز متعدد درخواستوں کے دوران حالت کو برقرار رکھتے ہیں، مسلسل بات چیت، سیاق و سباق کے جمع ہونے، اور وسائل کے انتظام کی حمایت کرتے ہیں۔
- **صلاحیتوں پر مذاکرات**: کنکشن سیٹ اپ کے دوران، کلائنٹس اور سرورز اپنی سپورٹ کردہ خصوصیات، پروٹوکول ورژنز، دستیاب ٹولز، اور وسائل کے بارے میں معلومات کا تبادلہ کرتے ہیں۔ یہ اس بات کو یقینی بناتا ہے کہ دونوں طرفیں ایک دوسرے کی صلاحیتوں کو سمجھیں اور مطابق ڈھال سکیں۔

#### ➕ اضافی یوٹیلٹیز

نیچے کچھ اضافی یوٹیلٹیز اور پروٹوکول ایکسٹینشنز دی گئی ہیں جو MCP ڈیولپر کے تجربے کو بہتر بنانے اور پیچیدہ منظرناموں کو ممکن بنانے کے لیے فراہم کرتا ہے:

- **کنفیگریشن آپشنز**: MCP سیشن پیرا میٹرز کی متحرک کنفیگریشن کی اجازت دیتا ہے، جیسے ٹول کی اجازتیں، وسائل تک رسائی، اور ماڈل کی ترتیبات، جو ہر تعامل کے مطابق ہوتی ہیں۔
- **پروگریس ٹریکنگ**: طویل چلنے والے آپریشنز پروگریس اپڈیٹس رپورٹ کر سکتے ہیں، جو پیچیدہ کاموں کے دوران ردعمل دینے والے صارف انٹرفیس اور بہتر صارف تجربہ کو ممکن بناتا ہے۔
- **درخواست منسوخی**: کلائنٹس چلتی ہوئی درخواستوں کو منسوخ کر سکتے ہیں، جس سے صارفین کو غیر ضروری یا طویل چلنے والی کارروائیوں کو روکنے کی سہولت ملتی ہے۔
- **ایرر رپورٹنگ**: معیاری ایرر پیغامات اور کوڈز مسائل کی تشخیص، ناکامیوں کو نرم انداز میں سنبھالنے، اور صارفین اور ڈیولپرز کو قابل عمل فیڈ بیک فراہم کرنے میں مدد کرتے ہیں۔
- **لاگنگ**: کلائنٹس اور سرورز دونوں آڈٹ، ڈیبگنگ، اور پروٹوکول تعاملات کی نگرانی کے لیے منظم لاگز جاری کر سکتے ہیں۔

ان پروٹوکول خصوصیات کا فائدہ اٹھاتے ہوئے، MCP زبان کے ماڈلز اور بیرونی ٹولز یا ڈیٹا ذرائع کے درمیان مضبوط، محفوظ، اور لچکدار مواصلات کو یقینی بناتا ہے۔

### 🔐 سیکیورٹی کے پہلو

MCP کے نفاذ کو محفوظ اور قابل اعتماد تعاملات کو یقینی بنانے کے لیے کئی کلیدی سیکیورٹی اصولوں پر عمل کرنا چاہیے:

- **صارف کی رضامندی اور کنٹرول**: کسی بھی ڈیٹا تک رسائی یا آپریشن انجام دینے سے پہلے صارفین کی واضح رضامندی ضروری ہے۔ انہیں اس بات کا واضح کنٹرول ہونا چاہیے کہ کون سا ڈیٹا شیئر کیا جا رہا ہے اور کون سی کارروائیاں مجاز ہیں، اور ان سرگرمیوں کا جائزہ لینے اور منظوری دینے کے لیے آسان صارف انٹرفیس فراہم کیے جائیں۔

- **ڈیٹا کی رازداری**: صارف کا ڈیٹا صرف واضح رضامندی کے ساتھ ظاہر کیا جانا چاہیے اور مناسب رسائی کنٹرولز کے ذریعے محفوظ ہونا چاہیے۔ MCP کے نفاذ کو غیر مجاز ڈیٹا کی منتقلی سے بچانا چاہیے اور تمام تعاملات کے دوران رازداری کو برقرار رکھنا چاہیے۔

- **ٹول کی حفاظت**: کسی بھی ٹول کو چلانے سے پہلے واضح صارف کی رضامندی ضروری ہے۔ صارفین کو ہر ٹول کی فعالیت کی واضح سمجھ ہونی چاہیے، اور مضبوط سیکیورٹی حدود نافذ کی جانی چاہئیں تاکہ غیر ارادی یا غیر محفوظ ٹول کے عمل کو روکا جا سکے۔

ان اصولوں پر عمل کرتے ہوئے، MCP اس بات کو یقینی بناتا ہے کہ صارف کا اعتماد، رازداری، اور حفاظت تمام پروٹوکول تعاملات میں برقرار رہے۔
