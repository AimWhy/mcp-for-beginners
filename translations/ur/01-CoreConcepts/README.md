# 📖 MCP کور تصورات: AI انضمام کے لیے ماڈل کانٹیکسٹ پروٹوکول میں مہارت حاصل کرنا

ماڈل کانٹیکسٹ پروٹوکول (MCP) ایک طاقتور، معیاری فریم ورک ہے جو بڑے زبان کے ماڈلز (LLMs) اور بیرونی ٹولز، ایپلیکیشنز، اور ڈیٹا ذرائع کے درمیان مواصلات کو بہتر بناتا ہے۔ یہ SEO-اپٹمائزڈ گائیڈ آپ کو MCP کے بنیادی تصورات سے روشناس کرائے گا، تاکہ آپ اس کے کلائنٹ-سرور آرکیٹیکچر، اہم اجزاء، مواصلاتی طریقہ کار، اور بہترین عمل درآمد کی سمجھ حاصل کر سکیں۔

## جائزہ

یہ سبق ماڈل کانٹیکسٹ پروٹوکول (MCP) کے ماحولیاتی نظام کی بنیادی آرکیٹیکچر اور اجزاء کا جائزہ لیتا ہے۔ آپ کلائنٹ-سرور آرکیٹیکچر، اہم اجزاء، اور MCP تعاملات کو چلانے والے مواصلاتی طریقوں کے بارے میں سیکھیں گے۔

## 👩‍🎓 کلیدی سیکھنے کے مقاصد

اس سبق کے اختتام تک، آپ:

- MCP کلائنٹ-سرور آرکیٹیکچر کو سمجھیں گے۔
- Hosts، Clients، اور Servers کے کردار اور ذمہ داریوں کی شناخت کریں گے۔
- MCP کو ایک لچکدار انضمام پرت بنانے والی بنیادی خصوصیات کا تجزیہ کریں گے۔
- MCP ماحولیاتی نظام میں معلومات کے بہاؤ کو سمجھیں گے۔
- .NET، Java، Python، اور JavaScript میں کوڈ مثالوں کے ذریعے عملی بصیرت حاصل کریں گے۔

## 🔎 MCP آرکیٹیکچر: ایک گہرا جائزہ

MCP کا ماحولیاتی نظام کلائنٹ-سرور ماڈل پر مبنی ہے۔ یہ ماڈیولر ڈھانچہ AI ایپلیکیشنز کو ٹولز، ڈیٹا بیسز، APIs، اور سیاق و سباق کے وسائل کے ساتھ مؤثر طریقے سے تعامل کرنے کی اجازت دیتا ہے۔ آئیے اس آرکیٹیکچر کو اس کے بنیادی اجزاء میں تقسیم کرتے ہیں۔

### 1. Hosts

ماڈل کانٹیکسٹ پروٹوکول (MCP) میں، Hosts ایک اہم کردار ادا کرتے ہیں کیونکہ یہ وہ بنیادی انٹرفیس ہیں جن کے ذریعے صارفین پروٹوکول کے ساتھ تعامل کرتے ہیں۔ Hosts وہ ایپلیکیشنز یا ماحول ہیں جو MCP سرورز کے ساتھ کنکشن شروع کرتے ہیں تاکہ ڈیٹا، ٹولز، اور پرامپٹس تک رسائی حاصل کی جا سکے۔ Hosts کی مثالوں میں Visual Studio Code جیسے انٹیگریٹڈ ڈیولپمنٹ ماحول (IDEs)، Claude Desktop جیسے AI ٹولز، یا مخصوص کاموں کے لیے بنائے گئے کسٹم ایجنٹس شامل ہیں۔

**Hosts** وہ LLM ایپلیکیشنز ہیں جو کنکشن شروع کرتے ہیں۔ یہ:

- AI ماڈلز کے ساتھ تعامل کرتے ہیں یا جوابات پیدا کرتے ہیں۔
- MCP سرورز کے ساتھ کنکشن شروع کرتے ہیں۔
- گفتگو کے بہاؤ اور یوزر انٹرفیس کا انتظام کرتے ہیں۔
- اجازت اور سیکیورٹی پابندیوں کو کنٹرول کرتے ہیں۔
- ڈیٹا شیئرنگ اور ٹول کے نفاذ کے لیے صارف کی رضامندی کو ہینڈل کرتے ہیں۔

### 2. Clients

Clients وہ ضروری اجزاء ہیں جو Hosts اور MCP سرورز کے درمیان تعامل کو آسان بناتے ہیں۔ Clients ثالث کا کردار ادا کرتے ہیں، جس سے Hosts کو MCP سرورز کی فراہم کردہ فعالیت تک رسائی اور استعمال کی سہولت ملتی ہے۔ یہ MCP آرکیٹیکچر میں ہموار مواصلات اور مؤثر ڈیٹا تبادلے کو یقینی بنانے میں اہم کردار ادا کرتے ہیں۔

**Clients** host ایپلیکیشن کے اندر کنیکٹرز ہوتے ہیں۔ یہ:

- سرورز کو پرامپٹس/ہدایات کے ساتھ درخواستیں بھیجتے ہیں۔
- سرورز کے ساتھ صلاحیتوں پر بات چیت کرتے ہیں۔
- ماڈلز سے ٹول نفاذ کی درخواستوں کا انتظام کرتے ہیں۔
- جوابات کو پروسیس اور صارفین کو دکھاتے ہیں۔

### 3. Servers

Servers MCP کلائنٹس کی درخواستوں کو سنبھالتے ہیں اور مناسب جوابات فراہم کرتے ہیں۔ یہ مختلف آپریشنز جیسے ڈیٹا بازیافت، ٹول نفاذ، اور پرامپٹ جنریشن کو منظم کرتے ہیں۔ Servers اس بات کو یقینی بناتے ہیں کہ کلائنٹس اور Hosts کے درمیان مواصلات مؤثر اور قابل اعتماد ہوں، اور تعامل کے عمل کی سالمیت برقرار رہے۔

**Servers** خدمات ہیں جو کانٹیکسٹ اور صلاحیتیں فراہم کرتی ہیں۔ یہ:

- دستیاب خصوصیات (وسائل، پرامپٹس، ٹولز) کو رجسٹر کرتے ہیں۔
- کلائنٹ سے ٹول کالز وصول کرتے اور انہیں انجام دیتے ہیں۔
- ماڈل کے جوابات کو بہتر بنانے کے لیے سیاق و سباق کی معلومات فراہم کرتے ہیں۔
- نتائج کلائنٹ کو واپس بھیجتے ہیں۔
- ضرورت پڑنے پر تعاملات کے دوران حالت کو برقرار رکھتے ہیں۔

Servers کو کوئی بھی تیار کر سکتا ہے تاکہ ماڈل کی صلاحیتوں کو مخصوص فنکشنلٹی کے ساتھ بڑھایا جا سکے۔

### 4. Server Features

Model Context Protocol (MCP) میں Servers بنیادی بلاکس فراہم کرتے ہیں جو کلائنٹس، Hosts، اور زبان کے ماڈلز کے درمیان بھرپور تعاملات کو ممکن بناتے ہیں۔ یہ خصوصیات MCP کی صلاحیتوں کو منظم کانٹیکسٹ، ٹولز، اور پرامپٹس فراہم کر کے بڑھاتی ہیں۔

MCP سرورز درج ذیل خصوصیات فراہم کر سکتے ہیں:

#### 📑 Resources

Model Context Protocol (MCP) میں Resources مختلف قسم کے کانٹیکسٹ اور ڈیٹا کو شامل کرتے ہیں جنہیں صارفین یا AI ماڈلز استعمال کر سکتے ہیں۔ ان میں شامل ہیں:

- **سیاق و سباق کا ڈیٹا**: معلومات اور کانٹیکسٹ جو صارفین یا AI ماڈلز فیصلہ سازی اور کام انجام دینے کے لیے استعمال کر سکتے ہیں۔
- **نالج بیسز اور دستاویزی ذخائر**: منظم اور غیر منظم ڈیٹا کے مجموعے، جیسے مضامین، دستی، اور تحقیقی مقالے، جو قیمتی بصیرت اور معلومات فراہم کرتے ہیں۔
- **مقامی فائلز اور ڈیٹا بیسز**: ڈیٹا جو مقامی طور پر آلات پر یا ڈیٹا بیسز میں ذخیرہ ہوتا ہے، جس تک رسائی حاصل کی جا سکتی ہے۔
- **APIs اور ویب سروسز**: بیرونی انٹرفیسز اور خدمات جو اضافی ڈیٹا اور فعالیت فراہم کرتی ہیں، مختلف آن لائن وسائل اور ٹولز کے ساتھ انضمام کی اجازت دیتی ہیں۔

ایک resource کی مثال ڈیٹا بیس اسکیمہ یا فائل ہو سکتی ہے جس تک اس طرح رسائی حاصل کی جا سکتی ہے:

```text
file://log.txt
database://schema
```

### 🤖 Prompts

Model Context Protocol (MCP) میں Prompts مختلف پیشگی متعین ٹیمپلیٹس اور تعامل کے نمونے شامل ہوتے ہیں جو صارف کے ورک فلو کو آسان بنانے اور مواصلات کو بہتر بنانے کے لیے بنائے گئے ہیں۔ ان میں شامل ہیں:

- **ٹیمپلیٹڈ پیغامات اور ورک فلو**: پیشگی ساختہ پیغامات اور عمل جو صارفین کو مخصوص کاموں اور تعاملات میں رہنمائی فراہم کرتے ہیں۔
- **پیشگی متعین تعامل کے نمونے**: معیاری کارروائیوں اور جوابات کے سلسلے جو مستقل اور مؤثر مواصلات کو آسان بناتے ہیں۔
- **خصوصی گفتگو کے ٹیمپلیٹس**: مخصوص قسم کی گفتگو کے لیے حسب ضرورت ٹیمپلیٹس، جو متعلقہ اور سیاق و سباق کے مطابق تعاملات کو یقینی بناتے ہیں۔

ایک پرامپٹ ٹیمپلیٹ اس طرح دکھائی دے سکتا ہے:

```markdown
Generate a product slogan based on the following {{product}} with the following {{keywords}}
```

#### ⛏️ Tools

Model Context Protocol (MCP) میں Tools وہ فنکشنز ہیں جنہیں AI ماڈل مخصوص کام انجام دینے کے لیے چلا سکتا ہے۔ یہ ٹولز AI ماڈل کی صلاحیتوں کو منظم اور قابل اعتماد آپریشنز فراہم کر کے بڑھاتے ہیں۔ اہم پہلو یہ ہیں:

- **AI ماڈل کے لیے چلانے والے فنکشنز**: ٹولز قابل عمل فنکشنز ہوتے ہیں جنہیں AI ماڈل مختلف کاموں کے لیے کال کر سکتا ہے۔
- **منفرد نام اور وضاحت**: ہر ٹول کا ایک منفرد نام اور تفصیلی وضاحت ہوتی ہے جو اس کے مقصد اور فعالیت کو بیان کرتی ہے۔
- **پیرا میٹرز اور آؤٹ پٹ**: ٹولز مخصوص پیرا میٹرز قبول کرتے ہیں اور منظم آؤٹ پٹ دیتے ہیں، جو مستقل اور متوقع نتائج کو یقینی بناتے ہیں۔
- **مخصوص فنکشنز**: ٹولز ویب سرچز، کیلکولیشنز، اور ڈیٹا بیس کوئریز جیسے الگ الگ فنکشنز انجام دیتے ہیں۔

ایک مثال کے طور پر ٹول اس طرح ہو سکتا ہے:

```typescript
server.tool(
  "GetProducts"
  {
    pageSize: z.string().optional(),
    pageCount: z.string().optional()
  }, () => {
    // return results from API
  }
)
```

## Client Features

Model Context Protocol (MCP) میں، clients سرورز کو کئی کلیدی خصوصیات فراہم کرتے ہیں، جو پروٹوکول کے اندر مجموعی فعالیت اور تعامل کو بڑھاتی ہیں۔ ان خصوصیات میں سے ایک نمایاں خصوصیت Sampling ہے۔

### 👉 Sampling

- **سرور کی طرف سے شروع کیے گئے ایجنٹک رویے**: clients سرورز کو مخصوص کارروائیاں یا رویے خودکار طور پر شروع کرنے کی اجازت دیتے ہیں، جو نظام کی متحرک صلاحیتوں کو بڑھاتا ہے۔
- **Recursive LLM Interactions**: یہ خصوصیت بڑے زبان کے ماڈلز (LLMs) کے ساتھ تکراری تعاملات کی اجازت دیتی ہے، جس سے کاموں کی زیادہ پیچیدہ اور تکراری پروسیسنگ ممکن ہوتی ہے۔
- **اضافی ماڈل مکمل کرنے کی درخواست**: سرورز ماڈل سے اضافی مکملات طلب کر سکتے ہیں، تاکہ جوابات مکمل اور سیاق و سباق کے لحاظ سے متعلقہ ہوں۔

## MCP میں معلومات کا بہاؤ

Model Context Protocol (MCP) میزبانوں، کلائنٹس، سرورز، اور ماڈلز کے درمیان معلومات کے منظم بہاؤ کی تعریف کرتا ہے۔ اس بہاؤ کو سمجھنا اس بات کو واضح کرتا ہے کہ صارف کی درخواستیں کیسے پروسیس ہوتی ہیں اور بیرونی ٹولز اور ڈیٹا ماڈلز کے جوابات میں کیسے ضم ہوتے ہیں۔

- **Host کنکشن شروع کرتا ہے**  
  میزبان ایپلیکیشن (جیسے IDE یا چیٹ انٹرفیس) عام طور پر STDIO، WebSocket، یا کسی دوسرے سپورٹڈ ٹرانسپورٹ کے ذریعے MCP سرور سے کنکشن قائم کرتا ہے۔

- **صلاحیتوں پر بات چیت**  
  کلائنٹ (جو میزبان میں ایمبیڈ ہوتا ہے) اور سرور اپنی سپورٹڈ خصوصیات، ٹولز، وسائل، اور پروٹوکول ورژنز کے بارے میں معلومات کا تبادلہ کرتے ہیں۔ اس سے دونوں طرف کو معلوم ہوتا ہے کہ سیشن کے لیے کون سی صلاحیتیں دستیاب ہیں۔

- **صارف کی درخواست**  
  صارف میزبان کے ساتھ تعامل کرتا ہے (مثلاً پرامپٹ یا کمانڈ داخل کرتا ہے)۔ میزبان اس ان پٹ کو جمع کرتا ہے اور پروسیسنگ کے لیے کلائنٹ کو بھیجتا ہے۔

- **وسائل یا ٹول کا استعمال**  
  - کلائنٹ سرور سے اضافی کانٹیکسٹ یا وسائل کی درخواست کر سکتا ہے (جیسے فائلز، ڈیٹا بیس اندراجات، یا نالج بیس آرٹیکلز) تاکہ ماڈل کی سمجھ بوجھ کو بہتر بنایا جا سکے۔
  - اگر ماڈل یہ طے کرتا ہے کہ ٹول کی ضرورت ہے (مثلاً ڈیٹا حاصل کرنے، کیلکولیشن کرنے، یا API کال کرنے کے لیے)، تو کلائنٹ سرور کو ٹول کال کی درخواست بھیجتا ہے، جس میں ٹول کا نام اور پیرا میٹرز شامل ہوتے ہیں۔

- **سرور کا نفاذ**  
  سرور وسائل یا ٹول کی درخواست وصول کرتا ہے، ضروری آپریشنز انجام دیتا ہے (جیسے فنکشن چلانا، ڈیٹا بیس سے پوچھنا، یا فائل بازیافت کرنا)، اور نتائج کو منظم شکل میں کلائنٹ کو واپس بھیجتا ہے۔

- **جواب کی تخلیق**  
  کلائنٹ سرور کے جوابات (وسائل کا ڈیٹا، ٹول کے آؤٹ پٹ وغیرہ) کو جاری ماڈل تعامل میں ضم کرتا ہے۔ ماڈل اس معلومات کو استعمال کرتے ہوئے جامع اور سیاق و سباق کے لحاظ سے متعلقہ جواب پیدا کرتا ہے۔

- **نتائج کی پیشکش**  
  میزبان کلائنٹ سے حتمی آؤٹ پٹ وصول کرتا ہے اور اسے صارف کو پیش کرتا ہے، عام طور پر ماڈل کی پیدا کردہ تحریر اور ٹول کے نفاذ یا وسائل کی تلاش کے نتائج دونوں شامل ہوتے ہیں۔

یہ بہاؤ MCP کو جدید، تعاملی، اور سیاق و سباق سے آگاہ AI ایپلیکیشنز کی حمایت کرنے کے قابل بناتا ہے، جو ماڈلز کو بیرونی ٹولز اور ڈیٹا ذرائع کے ساتھ بغیر رکاوٹ جوڑتا ہے۔

## پروٹوکول کی تفصیلات

MCP (Model Context Protocol) [JSON-RPC 2.0](https://www.jsonrpc.org/) پر مبنی ہے، جو میزبانوں، کلائنٹس، اور سرورز کے درمیان مواصلات کے لیے ایک معیاری، زبان سے آزاد میسج فارمیٹ فراہم کرتا ہے۔ یہ بنیاد مختلف پلیٹ فارمز اور پروگرامنگ زبانوں میں قابل اعتماد، منظم، اور قابل توسیع تعاملات کو ممکن بناتی ہے۔

### کلیدی پروٹوکول خصوصیات

MCP JSON-RPC 2.0 کو ٹول کال، وسائل تک رسائی، اور پرامپٹ مینجمنٹ کے اضافی اصولوں کے ساتھ بڑھاتا ہے۔ یہ متعدد ٹرانسپورٹ لیئرز (STDIO، WebSocket، SSE) کی حمایت کرتا ہے اور اجزاء کے درمیان محفوظ، قابل توسیع، اور زبان سے آزاد مواصلات کو فعال بناتا ہے۔

#### 🧢 بنیادی پروٹوکول

- **JSON-RPC میسج فارمیٹ**: تمام درخواستیں اور جوابات JSON-RPC 2.0 وضاحت استعمال کرتے ہیں، جو طریقہ کالز، پیرا میٹرز، نتائج، اور ایرر ہینڈلنگ کے لیے مستقل ڈھانچہ یقینی بناتا ہے۔
- **حالت دار کنکشنز**: MCP سیشن متعدد درخواستوں میں حالت کو برقرار رکھتے ہیں، جاری گفتگو، کانٹیکسٹ جمع کرنے، اور وسائل کے انتظام کی حمایت کرتے ہیں۔
- **صلاحیتوں پر بات چیت**: کنکشن سیٹ اپ کے دوران، کلائنٹس اور سرورز اپنی سپورٹڈ خصوصیات، پروٹوکول ورژنز، دستیاب ٹولز، اور وسائل کے بارے میں معلومات کا تبادلہ کرتے ہیں۔ اس سے دونوں طرف کو ایک دوسرے کی صلاحیتوں کا علم ہوتا ہے اور وہ مطابق ڈھل سکتے ہیں۔

#### ➕ اضافی یوٹیلٹیز

ذیل میں کچھ اضافی یوٹیلٹیز اور پروٹوکول توسیعات ہیں جو MCP فراہم کرتا ہے تاکہ ڈویلپر کے تجربے کو بہتر بنایا جا سکے اور جدید حالات کو فعال کیا جا سکے:

- **تشکیلاتی اختیارات**: MCP سیشن پیرامیٹرز کی متحرک ترتیب کی اجازت دیتا ہے، جیسے ٹول اجازتیں، وسائل تک رسائی، اور ماڈل کی ترتیبات، جو ہر تعامل کے لیے مخصوص ہوتی ہیں۔
- **پیش رفت کی ٹریکنگ**: طویل مدتی آپریشنز پیش رفت کی اپ ڈیٹس رپورٹ کر سکتے ہیں، جو پیچیدہ کاموں کے دوران جوابدہ یوزر انٹرفیس اور بہتر صارف تجربہ فراہم کرتا ہے۔
- **درخواست کی منسوخی**: کلائنٹس فلائٹ میں درخواستوں کو منسوخ کر سکتے ہیں، جس سے صارفین غیر ضروری یا بہت زیادہ وقت لینے والی کارروائیوں کو روک سکتے ہیں۔
- **خرابی کی رپورٹنگ**: معیاری خرابی کے پیغامات اور کوڈز مسائل کی تشخیص، ناکامیوں کو نرمی سے سنبھالنے، اور صارفین اور ڈویلپرز کو عملی فیڈ بیک فراہم کرنے میں مدد کرتے ہیں۔
- **لاگنگ**: کلائنٹس اور سرورز دونوں آڈٹ، ڈیبگنگ، اور پروٹوکول تعاملات کی نگرانی کے لیے منظم لاگز جاری کر سکتے ہیں۔

ان پروٹوکول خصوصیات کا فائدہ اٹھا کر، MCP زبان کے ماڈلز اور بیرونی ٹولز یا ڈیٹا ذرائع کے درمیان مضبوط، محفوظ، اور لچکدار مواصلات کو یقینی بناتا ہے۔

### 🔐 سیکیورٹی کے پہلو

MCP کی تنصیبات کو محفوظ اور قابل اعتماد تعاملات کو یقینی بنانے کے لیے کئی اہم سیکیورٹی اصولوں کی پابندی کرنی چاہیے:

- **صارف کی رضامندی اور کنٹرول**: کسی بھی ڈیٹا تک رسائی یا آپریشن کرنے سے پہلے صارفین کی واضح رضامندی ضروری ہے۔ انہیں یہ واضح کنٹرول ہونا چاہیے کہ کون سا ڈیٹا شیئر کیا جا رہا ہے اور کون سے اقدامات کی اجازت دی گئی ہے، جس کے لیے جائزہ لینے اور منظوری دینے کے لیے آسان صارف انٹرفیس فراہم کیے جائیں۔

- **ڈیٹا پرائیویسی**: صارف کا ڈیٹا صرف واضح رضامندی کے ساتھ ظاہر کیا جانا چاہیے اور مناسب رسائی کنٹرولز کے ذریعے محفوظ ہونا چاہیے۔ MCP کی تنصیبات کو غیر مجاز ڈیٹا ٹرانسمیشن سے بچانا چاہیے اور تمام تعاملات میں پرائیویسی کو برقرار رکھنا چاہیے۔

- **ٹول کی حفاظت**: کسی بھی ٹول کو کال کرنے سے پہلے صارف کی واضح رضامندی ضروری ہے۔ صارفین کو ہر ٹول کی فعالیت کا واضح ادراک ہونا چاہیے، اور مضبوط سیکیورٹی حدود نافذ کی جانی چاہئیں تاکہ غیر ارادی یا غیر محفوظ ٹول نفاذ کو روکا جا سکے۔

ان اصولوں کی پیروی کر کے، MCP اس بات کو یقینی بناتا ہے کہ صارف کا اعتماد، پرائیویسی، اور حفاظت تمام پروٹوکول تعاملات میں برقرار رہے۔

## کوڈ مثالیں: کلیدی اجزاء

ذیل میں چند مقبول پروگرامنگ زبانوں میں کوڈ مثالیں دی گئی ہیں جو MCP سرور کے کلیدی اجزاء اور ٹولز کو نافذ کرنے کا طریقہ دکھاتی ہیں۔

### .NET مثال: Tools کے ساتھ ایک سادہ MCP سرور بنانا

**ڈس کلیمر**:  
یہ دستاویز AI ترجمہ سروس [Co-op Translator](https://github.com/Azure/co-op-translator) کے ذریعے ترجمہ کی گئی ہے۔ اگرچہ ہم درستگی کے لیے کوشاں ہیں، براہ کرم نوٹ کریں کہ خودکار ترجمے میں غلطیاں یا بے ضابطگیاں ہو سکتی ہیں۔ اصل دستاویز اپنی مادری زبان میں معتبر ماخذ سمجھی جانی چاہیے۔ اہم معلومات کے لیے پیشہ ور انسانی ترجمہ تجویز کیا جاتا ہے۔ ہم اس ترجمے کے استعمال سے پیدا ہونے والی کسی بھی غلط فہمی یا غلط تشریح کے ذمہ دار نہیں ہیں۔
