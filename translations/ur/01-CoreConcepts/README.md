# 📖 MCP Core Concepts: ماڈل کانٹیکسٹ پروٹوکول برائے AI انٹیگریشن میں مہارت

Model Context Protocol (MCP) ایک طاقتور، معیاری فریم ورک ہے جو Large Language Models (LLMs) اور بیرونی ٹولز، ایپلیکیشنز، اور ڈیٹا ذرائع کے درمیان مواصلات کو بہتر بناتا ہے۔ یہ SEO-اپٹمائزڈ گائیڈ آپ کو MCP کے بنیادی تصورات سے روشناس کرائے گا، تاکہ آپ اس کی کلائنٹ-سرور آرکیٹیکچر، اہم اجزاء، مواصلاتی طریقہ کار، اور بہترین عمل درآمد کو سمجھ سکیں۔

## جائزہ

یہ سبق Model Context Protocol (MCP) کے بنیادی آرکیٹیکچر اور اجزاء کو دریافت کرتا ہے جو MCP ایکو سسٹم کی تشکیل کرتے ہیں۔ آپ کلائنٹ-سرور آرکیٹیکچر، اہم اجزاء، اور مواصلاتی طریقہ کار کے بارے میں سیکھیں گے جو MCP تعاملات کو ممکن بناتے ہیں۔

## 👩‍🎓 کلیدی سیکھنے کے مقاصد

اس سبق کے اختتام پر آپ:

- MCP کلائنٹ-سرور آرکیٹیکچر کو سمجھیں گے۔
- Hosts, Clients, اور Servers کے کردار اور ذمہ داریاں پہچانیں گے۔
- MCP کو ایک لچکدار انٹیگریشن لیئر بنانے والی بنیادی خصوصیات کا تجزیہ کریں گے۔
- MCP ایکو سسٹم میں معلومات کے بہاؤ کو سمجھیں گے۔
- .NET, Java, Python, اور JavaScript میں کوڈ مثالوں کے ذریعے عملی بصیرت حاصل کریں گے۔

## 🔎 MCP آرکیٹیکچر: ایک گہرا جائزہ

MCP ایکو سسٹم کلائنٹ-سرور ماڈل پر مبنی ہے۔ یہ ماڈیولر ڈھانچہ AI ایپلیکیشنز کو ٹولز، ڈیٹا بیسز، APIs، اور کانٹیکسچول وسائل کے ساتھ مؤثر طریقے سے تعامل کرنے کی اجازت دیتا ہے۔ آئیے اس آرکیٹیکچر کو اس کے بنیادی اجزاء میں تقسیم کرتے ہیں۔

### 1. Hosts

Model Context Protocol (MCP) میں Hosts ایک اہم کردار ادا کرتے ہیں کیونکہ یہ وہ بنیادی انٹرفیس ہوتے ہیں جن کے ذریعے صارفین پروٹوکول کے ساتھ تعامل کرتے ہیں۔ Hosts وہ ایپلیکیشنز یا ماحولیات ہیں جو MCP سرورز کے ساتھ کنکشن شروع کرتے ہیں تاکہ ڈیٹا، ٹولز، اور پرامپٹس تک رسائی حاصل کی جا سکے۔ Hosts کی مثالوں میں Visual Studio Code جیسے Integrated Development Environments (IDEs)، Claude Desktop جیسے AI ٹولز، یا مخصوص کاموں کے لیے بنائے گئے کسٹم ایجنٹس شامل ہیں۔

**Hosts** LLM ایپلیکیشنز ہیں جو کنکشن شروع کرتے ہیں۔ وہ:

- AI ماڈلز کے ساتھ تعامل کرتے ہیں تاکہ جوابات تیار کریں۔
- MCP سرورز کے ساتھ کنکشن شروع کرتے ہیں۔
- گفتگو کے بہاؤ اور یوزر انٹرفیس کو مینیج کرتے ہیں۔
- اجازت اور سیکیورٹی پابندیوں کو کنٹرول کرتے ہیں۔
- ڈیٹا شیئرنگ اور ٹولز کے نفاذ کے لیے صارف کی رضامندی کو ہینڈل کرتے ہیں۔

### 2. Clients

Clients وہ اہم اجزاء ہیں جو Hosts اور MCP سرورز کے درمیان تعامل کو آسان بناتے ہیں۔ Clients درمیانی کردار ادا کرتے ہیں، جس سے Hosts MCP سرورز کی فراہم کردہ فعالیت تک رسائی حاصل کر سکتے ہیں۔ وہ MCP آرکیٹیکچر میں ہموار مواصلات اور مؤثر ڈیٹا تبادلے کو یقینی بناتے ہیں۔

**Clients** host ایپلیکیشن کے اندر کنیکٹرز ہیں۔ وہ:

- پرامپٹس/ہدایات کے ساتھ سرورز کو درخواستیں بھیجتے ہیں۔
- سرورز کے ساتھ صلاحیتوں پر مذاکرات کرتے ہیں۔
- ماڈلز سے ٹولز کے نفاذ کی درخواستوں کو مینیج کرتے ہیں۔
- صارفین کو جوابات پروسیس اور دکھاتے ہیں۔

### 3. Servers

Servers MCP کلائنٹس کی درخواستوں کو ہینڈل کرتے ہیں اور مناسب جوابات فراہم کرتے ہیں۔ وہ مختلف آپریشنز جیسے ڈیٹا بازیافت، ٹولز کا نفاذ، اور پرامپٹ جنریشن کو مینیج کرتے ہیں۔ Servers اس بات کو یقینی بناتے ہیں کہ کلائنٹس اور Hosts کے درمیان مواصلات مؤثر اور قابل اعتماد ہوں، اور تعامل کے عمل کی سالمیت برقرار رکھیں۔

**Servers** وہ خدمات ہیں جو کانٹیکسٹ اور صلاحیتیں فراہم کرتی ہیں۔ وہ:

- دستیاب خصوصیات (وسائل، پرامپٹس، ٹولز) کو رجسٹر کرتے ہیں۔
- کلائنٹ سے ٹول کالز وصول کرتے اور انہیں انجام دیتے ہیں۔
- ماڈل جوابات کو بہتر بنانے کے لیے کانٹیکسچول معلومات فراہم کرتے ہیں۔
- نتائج واپس کلائنٹ کو بھیجتے ہیں۔
- ضرورت پڑنے پر تعاملات کے دوران حالت برقرار رکھتے ہیں۔

Servers کو کوئی بھی تیار کر سکتا ہے تاکہ ماڈل کی صلاحیتوں کو خصوصی فعالیت کے ساتھ بڑھایا جا سکے۔

### 4. Server Features

Model Context Protocol (MCP) میں Servers بنیادی بلاکس فراہم کرتے ہیں جو کلائنٹس، Hosts، اور زبان کے ماڈلز کے درمیان بھرپور تعاملات کو ممکن بناتے ہیں۔ یہ خصوصیات MCP کی صلاحیتوں کو بہتر بنانے کے لیے ڈیزائن کی گئی ہیں، جو منظم کانٹیکسٹ، ٹولز، اور پرامپٹس فراہم کرتی ہیں۔

MCP سرورز درج ذیل خصوصیات پیش کر سکتے ہیں:

#### 📑 Resources

Model Context Protocol (MCP) میں Resources مختلف اقسام کے کانٹیکسٹ اور ڈیٹا پر مشتمل ہوتے ہیں جنہیں صارفین یا AI ماڈلز استعمال کر سکتے ہیں۔ ان میں شامل ہیں:

- **کانٹیکسچول ڈیٹا**: معلومات اور سیاق و سباق جو صارفین یا AI ماڈلز فیصلہ سازی اور کام انجام دینے کے لیے استعمال کر سکتے ہیں۔
- **نالج بیسز اور ڈاکیومنٹ ریپوزیٹریز**: منظم اور غیر منظم ڈیٹا کا مجموعہ، جیسے مضامین، مینولز، اور تحقیقی مقالے، جو قیمتی بصیرت اور معلومات فراہم کرتے ہیں۔
- **مقامی فائلیں اور ڈیٹا بیسز**: ڈیٹا جو آلات پر یا ڈیٹا بیسز میں مقامی طور پر محفوظ ہوتا ہے، جسے پراسیسنگ اور تجزیہ کے لیے حاصل کیا جا سکتا ہے۔
- **APIs اور ویب سروسز**: بیرونی انٹرفیسز اور خدمات جو اضافی ڈیٹا اور فعالیت فراہم کرتی ہیں، اور مختلف آن لائن وسائل اور ٹولز کے ساتھ انضمام کی اجازت دیتی ہیں۔

وسیلے کی مثال ایک ڈیٹا بیس اسکیمہ یا فائل ہو سکتی ہے جس تک اس طرح رسائی حاصل کی جا سکتی ہے:

```text
file://log.txt
database://schema
```

### 🤖 Prompts

Model Context Protocol (MCP) میں پرامپٹس مختلف پیشگی طے شدہ ٹیمپلیٹس اور تعامل کے پیٹرنز شامل ہوتے ہیں جو صارف کے ورک فلو کو آسان بنانے اور مواصلات کو بہتر بنانے کے لیے ڈیزائن کیے گئے ہیں۔ ان میں شامل ہیں:

- **ٹیمپلیٹ شدہ پیغامات اور ورک فلو**: پیش ساختہ پیغامات اور عمل جو صارفین کو مخصوص کاموں اور تعاملات کے ذریعے رہنمائی کرتے ہیں۔
- **پیشگی طے شدہ تعاملاتی پیٹرنز**: ایک جیسے اور مؤثر مواصلات کے لیے معیاری عمل اور جوابات کے سلسلے۔
- **خصوصی گفتگو کے ٹیمپلیٹس**: مخصوص قسم کی گفتگو کے لیے حسب ضرورت ٹیمپلیٹس جو متعلقہ اور سیاق و سباق کے مطابق تعاملات کو یقینی بناتے ہیں۔

ایک پرامپٹ ٹیمپلیٹ اس طرح دکھائی دے سکتا ہے:

```markdown
Generate a product slogan based on the following {{product}} with the following {{keywords}}
```

#### ⛏️ Tools

Model Context Protocol (MCP) میں Tools وہ فنکشنز ہیں جو AI ماڈل مخصوص کام انجام دینے کے لیے چلا سکتا ہے۔ یہ ٹولز AI ماڈل کی صلاحیتوں کو بڑھانے کے لیے ڈیزائن کیے گئے ہیں تاکہ منظم اور قابل اعتماد آپریشنز فراہم کریں۔ اہم پہلو شامل ہیں:

- **AI ماڈل کے لیے چلائے جانے والے فنکشنز**: ٹولز قابل عمل فنکشنز ہیں جنہیں AI ماڈل مختلف کام انجام دینے کے لیے بلا سکتا ہے۔
- **منفرد نام اور تفصیل**: ہر ٹول کا ایک منفرد نام اور مفصل وضاحت ہوتی ہے جو اس کے مقصد اور فعالیت کو بیان کرتی ہے۔
- **پیرامیٹرز اور آؤٹ پٹس**: ٹولز مخصوص پیرامیٹرز قبول کرتے ہیں اور منظم نتائج واپس کرتے ہیں تاکہ مستقل اور متوقع نتائج حاصل ہوں۔
- **مخصوص فنکشنز**: ٹولز مخصوص کام انجام دیتے ہیں جیسے ویب سرچ، حساب کتاب، اور ڈیٹا بیس کی تلاش۔

ایک ٹول کی مثال اس طرح ہو سکتی ہے:

```typescript
server.tool(
  "GetProducts",
  {
    pageSize: z.string().optional(),
    pageCount: z.string().optional()
  }, () => {
    // return results from API
  }
)
```

## Client Features

Model Context Protocol (MCP) میں کلائنٹس سرورز کو کئی کلیدی خصوصیات فراہم کرتے ہیں، جو پروٹوکول کے اندر مجموعی فعالیت اور تعامل کو بڑھاتی ہیں۔ ان خصوصیات میں سے ایک نمایاں خصوصیت Sampling ہے۔

### 👉 Sampling

- **سرور کی طرف سے خود مختار ایجنٹک رویے**: کلائنٹس سرورز کو مخصوص کارروائیاں یا رویے خود مختار طور پر شروع کرنے کی اجازت دیتے ہیں، جو نظام کی متحرک صلاحیتوں کو بڑھاتا ہے۔
- **Recursive LLM Interactions**: یہ خصوصیت بڑے زبان کے ماڈلز (LLMs) کے ساتھ recursive تعاملات کی اجازت دیتی ہے، جو زیادہ پیچیدہ اور تکراری کاموں کی پروسیسنگ ممکن بناتی ہے۔
- **اضافی ماڈل کمپلیشن کی درخواست**: سرورز ماڈل سے مزید کمپلیشنز کی درخواست کر سکتے ہیں تاکہ جوابات مکمل اور سیاق و سباق کے لحاظ سے متعلقہ ہوں۔

## MCP میں معلومات کا بہاؤ

Model Context Protocol (MCP) میزبان، کلائنٹس، سرورز، اور ماڈلز کے درمیان معلومات کے منظم بہاؤ کی وضاحت کرتا ہے۔ اس بہاؤ کو سمجھنا یہ واضح کرتا ہے کہ صارف کی درخواستیں کیسے پروسیس ہوتی ہیں اور بیرونی ٹولز اور ڈیٹا ماڈل کے جوابات میں کیسے شامل کیے جاتے ہیں۔

- **Host کنکشن شروع کرتا ہے**  
  میزبان ایپلیکیشن (جیسے IDE یا چیٹ انٹرفیس) MCP سرور سے کنکشن قائم کرتا ہے، عام طور پر STDIO، WebSocket، یا کسی دوسرے سپورٹڈ ٹرانسپورٹ کے ذریعے۔

- **صلاحیتوں پر مذاکرات**  
  کلائنٹ (جو میزبان میں شامل ہوتا ہے) اور سرور اپنی سپورٹ کردہ خصوصیات، ٹولز، وسائل، اور پروٹوکول ورژنز کے بارے میں معلومات کا تبادلہ کرتے ہیں۔ یہ یقینی بناتا ہے کہ دونوں طرف سیشن کے لیے دستیاب صلاحیتوں کو سمجھتے ہیں۔

- **صارف کی درخواست**  
  صارف میزبان کے ساتھ تعامل کرتا ہے (مثلاً پرامپٹ یا کمانڈ داخل کرتا ہے)۔ میزبان یہ ان پٹ جمع کرتا ہے اور پروسیسنگ کے لیے کلائنٹ کو بھیجتا ہے۔

- **وسیلے یا ٹول کا استعمال**  
  - کلائنٹ سرور سے اضافی کانٹیکسٹ یا وسائل کی درخواست کر سکتا ہے (جیسے فائلیں، ڈیٹا بیس انٹریز، یا نالج بیس آرٹیکلز) تاکہ ماڈل کی سمجھ کو بڑھایا جا سکے۔
  - اگر ماڈل کو ٹول کی ضرورت ہوتی ہے (مثلاً ڈیٹا حاصل کرنا، حساب کتاب کرنا، یا API کال کرنا)، تو کلائنٹ سرور کو ٹول کال کی درخواست بھیجتا ہے، جس میں ٹول کا نام اور پیرامیٹرز شامل ہوتے ہیں۔

- **سرور کا نفاذ**  
  سرور وسائل یا ٹول کی درخواست وصول کرتا ہے، ضروری آپریشنز انجام دیتا ہے (جیسے فنکشن چلانا، ڈیٹا بیس کو کوئری کرنا، یا فائل بازیافت کرنا)، اور منظم فارمیٹ میں نتائج کلائنٹ کو واپس بھیجتا ہے۔

- **جواب کی تخلیق**  
  کلائنٹ سرور کے جوابات (وسیلہ ڈیٹا، ٹول آؤٹ پٹس، وغیرہ) کو ماڈل کے تعامل میں شامل کرتا ہے۔ ماڈل اس معلومات کو استعمال کرتے ہوئے جامع اور سیاق و سباق کے لحاظ سے متعلقہ جواب تیار کرتا ہے۔

- **نتائج کی پیش کش**  
  میزبان کلائنٹ سے حتمی آؤٹ پٹ وصول کرتا ہے اور اسے صارف کو پیش کرتا ہے، جس میں اکثر ماڈل کا تیار کردہ متن اور ٹولز کے نفاذ یا وسائل کی تلاش کے نتائج شامل ہوتے ہیں۔

یہ بہاؤ MCP کو جدید، انٹرایکٹو، اور کانٹیکسٹ-آگاہ AI ایپلیکیشنز کی حمایت کرنے کے قابل بناتا ہے، جو ماڈلز کو بیرونی ٹولز اور ڈیٹا ذرائع سے باآسانی جوڑتا ہے۔

## پروٹوکول کی تفصیلات

MCP (Model Context Protocol) [JSON-RPC 2.0](https://www.jsonrpc.org/) کے اوپر بنایا گیا ہے، جو میزبانوں، کلائنٹس، اور سرورز کے درمیان مواصلات کے لیے ایک معیاری، زبان سے آزاد پیغام فارمیٹ فراہم کرتا ہے۔ یہ بنیاد مختلف پلیٹ فارمز اور پروگرامنگ زبانوں میں قابل اعتماد، منظم، اور توسیع پذیر تعاملات کو ممکن بناتی ہے۔

### کلیدی پروٹوکول خصوصیات

MCP JSON-RPC 2.0 کو اضافی کنونشنز کے ساتھ بڑھاتا ہے جو ٹول کالز، وسائل تک رسائی، اور پرامپٹ مینجمنٹ کے لیے ہیں۔ یہ متعدد ٹرانسپورٹ پرتوں (STDIO, WebSocket, SSE) کی حمایت کرتا ہے اور اجزاء کے درمیان محفوظ، توسیع پذیر، اور زبان سے آزاد مواصلات کو فعال بناتا ہے۔

#### 🧢 بنیادی پروٹوکول

- **JSON-RPC میسج فارمیٹ**: تمام درخواستیں اور جوابات JSON-RPC 2.0 وضاحت استعمال کرتے ہیں، جو میتھڈ کالز، پیرامیٹرز، نتائج، اور ایرر ہینڈلنگ کے لیے مستقل ڈھانچہ فراہم کرتا ہے۔
- **ریاستی کنکشنز**: MCP سیشنز متعدد درخواستوں کے دوران حالت برقرار رکھتے ہیں، گفتگو جاری رکھنے، کانٹیکسٹ اکٹھا کرنے، اور وسائل کے انتظام کی حمایت کرتے ہیں۔
- **صلاحیتوں پر مذاکرات**: کنکشن سیٹ اپ کے دوران، کلائنٹس اور سرورز اپنی سپورٹ کردہ خصوصیات، پروٹوکول ورژنز، دستیاب ٹولز، اور وسائل کے بارے میں معلومات کا تبادلہ کرتے ہیں۔ یہ یقینی بناتا ہے کہ دونوں طرف ایک دوسرے کی صلاحیتوں کو سمجھتے ہیں اور مطابق ڈھل سکتے ہیں۔

#### ➕ اضافی سہولیات

ذیل میں کچھ اضافی سہولیات اور پروٹوکول ایکسٹینشنز دی گئی ہیں جو MCP ڈویلپر کے تجربے کو بہتر بنانے اور پیچیدہ مناظر کی اجازت دینے کے لیے فراہم کرتا ہے:

- **کنفیگریشن آپشنز**: MCP سیشن پیرامیٹرز کو متحرک طور پر کنفیگر کرنے کی اجازت دیتا ہے، جیسے ٹول کی اجازتیں، وسائل تک رسائی، اور ماڈل کی ترتیبات، ہر تعامل کے مطابق۔
- **پروگریس ٹریکنگ**: طویل چلنے والے آپریشنز پیش رفت کی تازہ کاری دے سکتے ہیں، جو پیچیدہ کاموں کے دوران جوابدہ یوزر انٹرفیس اور بہتر یوزر تجربہ فراہم کرتا ہے۔
- **درخواست کی منسوخی**: کلائنٹس چلتی ہوئی درخواستوں کو منسوخ کر سکتے ہیں، جس سے صارفین غیر ضروری یا بہت طویل عمل کو روک سکتے ہیں۔
- **ایرر رپورٹنگ**: معیاری ایرر میسجز اور کوڈز مسائل کی تشخیص، خرابیوں کو مؤثر طریقے سے ہینڈل کرنے، اور صارفین و ڈویلپرز کو قابل عمل فیڈبیک فراہم کرنے میں مدد کرتے ہیں۔
- **لاگنگ**: کلائنٹس اور سرورز دونوں آڈٹ، ڈیبگنگ، اور پروٹوکول تعاملات کی مانیٹرنگ کے لیے منظم لاگز جاری کر سکتے ہیں۔

ان پروٹوکول خصوصیات کا فائدہ اٹھاتے ہوئے، MCP زبان کے ماڈلز اور بیرونی ٹولز یا ڈیٹا ذرائع کے درمیان مضبوط، محفوظ، اور لچکدار مواصلات کو یقینی بناتا ہے۔

### 🔐 سیکیورٹی کے پہلو

MCP کے نفاذ کو محفوظ اور قابل اعتماد تعاملات کو یقینی بنانے کے لیے چند کلیدی سیکیورٹی اصولوں پر عمل کرنا چاہیے:

- **صارف کی رضامندی اور کنٹرول**: کسی بھی ڈیٹا تک رسائی یا آپریشنز کرنے سے پہلے صارف کی واضح رضامندی ضروری ہے۔ صارفین کو واضح کنٹرول ہونا چاہیے کہ کون سا ڈیٹا شیئر کیا جاتا ہے اور کون سی کارروائیاں منظور کی گئی ہیں، جس کی حمایت آسان یوزر انٹرفیس سے ہو جو سرگرمیوں کا جائزہ لینے اور منظوری دینے میں مدد دے۔

- **ڈیٹا پرائیویسی**: صارف کا ڈیٹا صرف واضح رضامندی کے ساتھ ظاہر ہونا چاہیے اور مناسب رسائی کنٹرولز سے محفوظ ہونا چاہیے۔ MCP نفاذ کو غیر مجاز ڈیٹا ٹرانسمیشن سے بچانا چاہیے اور تمام تعاملات میں پرائیویسی کو برقرار رکھنا چاہیے۔

- **ٹول کی حفاظت**: کسی بھی ٹول کو چلانے سے پہلے صارف کی واضح رضامندی ضروری ہے۔ صارفین کو ہر ٹول کی فعالیت کی واضح سمجھ ہونی چاہیے، اور مضبوط سیکیورٹی حدود نافذ کی جانی چاہئیں تاکہ غیر ارادی یا غیر محفوظ ٹول نفاذ سے بچا جا سکے۔

ان اصولوں پر عمل کرتے ہوئے، MCP صارف کے اعتماد، پرائیویسی، اور حفاظت کو تمام پروٹوکول تعاملات میں برقرار رکھتا ہے۔
