ठिक छ, हाम्रो अर्को चरणका लागि, अब सर्भरमा रहेका क्षमताहरू सूचीबद्ध गरौं।

### -2 सर्भर क्षमताहरू सूचीबद्ध गर्नुहोस्

अब हामी सर्भरसँग जडान गर्नेछौं र यसको क्षमताहरू सोध्नेछौं:

### -3 सर्भर क्षमताहरूलाई LLM उपकरणहरूमा रूपान्तरण गर्नुहोस्

सर्भर क्षमताहरू सूचीबद्ध गरेपछि अर्को चरण ती क्षमताहरूलाई LLM ले बुझ्ने स्वरूपमा रूपान्तरण गर्नु हो। एक पटक हामीले यो गरिसकेपछि, यी क्षमताहरूलाई हाम्रो LLM का लागि उपकरणको रूपमा प्रदान गर्न सकिन्छ।

ठिक छ, अब हामी प्रयोगकर्ताका अनुरोधहरू ह्यान्डल गर्न तयार छौं, त्यसैले अब त्यसलाई सम्हाल्ने काम गरौं।

### -4 प्रयोगकर्ता प्रॉम्प्ट अनुरोध ह्यान्डल गर्नुहोस्

यस कोड भागमा, हामी प्रयोगकर्ताका अनुरोधहरू ह्यान्डल गर्नेछौं।

ठिक छ, तपाईंले यो गर्नुभयो!

## असाइनमेन्ट

अभ्यासबाट कोड लिएर थप उपकरणहरूसहित सर्भर निर्माण गर्नुहोस्। त्यसपछि अभ्यासमा जस्तै LLM सहितको क्लाइन्ट सिर्जना गर्नुहोस् र विभिन्न प्रॉम्प्टहरू प्रयोग गरेर परीक्षण गर्नुहोस् ताकि तपाईंका सबै सर्भर उपकरणहरू गतिशील रूपमा कल हुनेछन्। यसरी क्लाइन्ट निर्माण गर्दा अन्तिम प्रयोगकर्ताले उत्कृष्ट अनुभव पाउनेछ किनभने उनीहरू ठ्याक्कै क्लाइन्ट कमाण्डहरू नभई प्रॉम्प्टहरू प्रयोग गरेर कार्य गर्न सक्षम हुनेछन् र कुनै MCP सर्भर कल भइरहेको छ भन्ने थाहा नपाउनेछन्।
