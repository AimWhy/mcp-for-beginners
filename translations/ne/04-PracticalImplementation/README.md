# व्यावहारिक कार्यान्वयन

व्यावहारिक कार्यान्वयन भनेको Model Context Protocol (MCP) को शक्ति प्रत्यक्ष देखिने स्थान हो। MCP को सिद्धान्त र संरचनालाई बुझ्नु महत्वपूर्ण भए तापनि, साँच्चिकै मूल्य तब आउँछ जब तपाईं यी अवधारणाहरूलाई वास्तविक समस्याहरू समाधान गर्ने समाधानहरू निर्माण, परीक्षण, र तैनाथ गर्न प्रयोग गर्नुहुन्छ। यो अध्यायले वैचारिक ज्ञान र व्यावहारिक विकास बीचको खाडललाई पूर्ति गर्दछ र MCP-आधारित अनुप्रयोगहरूलाई जीवन्त बनाउने प्रक्रियामा तपाईंलाई मार्गदर्शन गर्दछ।

तपाईं बुद्धिमान सहायकहरू विकास गर्दै हुनुहुन्छ, व्यवसायिक कार्यप्रवाहहरूमा AI समावेश गर्दै हुनुहुन्छ वा डाटा प्रशोधनका लागि अनुकूलित उपकरणहरू निर्माण गर्दै हुनुहुन्छ भने पनि, MCP ले लचिलो आधार प्रदान गर्दछ। यसको भाषा-स्वतन्त्र डिजाइन र लोकप्रिय प्रोग्रामिङ भाषाहरूका लागि आधिकारिक SDK हरूले यसलाई विभिन्न विकासकर्ताहरूका लागि पहुँचयोग्य बनाउँछ। यी SDK हरूको उपयोग गरेर, तपाईं छिटो प्रोटोटाइप बनाउन, पुनरावृत्ति गर्न, र विभिन्न प्लेटफर्म तथा वातावरणहरूमा समाधानहरू विस्तार गर्न सक्नुहुन्छ।

अर्को भागहरूमा, तपाईंले C#, Java, TypeScript, JavaScript, र Python मा MCP कसरी कार्यान्वयन गर्ने भनेर देखाउने व्यावहारिक उदाहरणहरू, नमुना कोड, र तैनाथ रणनीतिहरू पाउनुहुनेछ। तपाईंले आफ्नो MCP सर्भरहरू कसरी डिबग र परीक्षण गर्ने, API हरू कसरी व्यवस्थापन गर्ने, र Azure प्रयोग गरेर क्लाउडमा समाधानहरू कसरी तैनाथ गर्ने सिक्नुहुनेछ। यी व्यावहारिक स्रोतहरूले तपाईंको सिकाइलाई तीव्र पार्ने र आत्मविश्वासका साथ बलियो, उत्पादन-तय MCP अनुप्रयोगहरू निर्माण गर्न सहयोग पुर्‍याउने उद्देश्य राख्छन्।

## अवलोकन

यो पाठले विभिन्न प्रोग्रामिङ भाषाहरूमा MCP कार्यान्वयनका व्यावहारिक पक्षहरूमा केन्द्रित छ। हामी C#, Java, TypeScript, JavaScript, र Python मा MCP SDK हरू कसरी प्रयोग गर्ने, MCP सर्भरहरू डिबग र परीक्षण गर्ने, र पुन: प्रयोग गर्न मिल्ने स्रोतहरू, प्रम्प्टहरू, र उपकरणहरू कसरी सिर्जना गर्ने भनेर अन्वेषण गर्नेछौं।

## सिकाइका उद्देश्यहरू

यस पाठको अन्त्यसम्म, तपाईं सक्षम हुनुहुनेछ:
- विभिन्न प्रोग्रामिङ भाषाहरूमा आधिकारिक SDK हरू प्रयोग गरेर MCP समाधानहरू कार्यान्वयन गर्न
- MCP सर्भरहरूलाई व्यवस्थित रूपमा डिबग र परीक्षण गर्न
- सर्भर सुविधाहरू (स्रोतहरू, प्रम्प्टहरू, र उपकरणहरू) सिर्जना र प्रयोग गर्न
- जटिल कार्यहरूको लागि प्रभावकारी MCP कार्यप्रवाहहरू डिजाइन गर्न
- प्रदर्शन र विश्वसनीयताका लागि MCP कार्यान्वयनहरू अनुकूलन गर्न

## मुख्य सर्भर सुविधाहरू

MCP सर्भरहरूले यी सुविधाहरूको कुनै पनि संयोजन कार्यान्वयन गर्न सक्छन्:

### स्रोतहरू
स्रोतहरूले प्रयोगकर्ता वा AI मोडेलले प्रयोग गर्ने सन्दर्भ र डाटा प्रदान गर्छन्:
- कागजात भण्डारहरू
- ज्ञान आधारहरू
- संरचित डाटा स्रोतहरू
- फाइल प्रणालीहरू

### प्रम्प्टहरू
प्रम्प्टहरू प्रयोगकर्ताका लागि टेम्प्लेट गरिएको सन्देश र कार्यप्रवाहहरू हुन्:
- पूर्वनिर्धारित संवाद टेम्प्लेटहरू
- मार्गनिर्देशित अन्तरक्रिया ढाँचाहरू
- विशेष संवाद संरचनाहरू

### उपकरणहरू
उपकरणहरू AI मोडेलले कार्यान्वयन गर्ने कार्यहरू हुन्:
- डाटा प्रशोधन उपयोगिताहरू
- बाह्य API एकीकरणहरू
- गणनात्मक क्षमता
- खोज कार्यक्षमता

## नमुना कार्यान्वयन: C#

आधिकारिक C# SDK रिपोजिटरीमा MCP का विभिन्न पक्षहरू देखाउने धेरै नमुना कार्यान्वयनहरू समावेश छन्:

- **मूल MCP क्लाइन्ट**: MCP क्लाइन्ट कसरी सिर्जना गर्ने र उपकरणहरू कसरी कल गर्ने सरल उदाहरण
- **मूल MCP सर्भर**: आधारभूत उपकरण दर्तासहित न्यूनतम सर्भर कार्यान्वयन
- **उन्नत MCP सर्भर**: उपकरण दर्ता, प्रमाणीकरण, र त्रुटि ह्यान्डलिङ सहित पूर्ण सुविधायुक्त सर्भर
- **ASP.NET एकीकरण**: ASP.NET Core सँगको एकीकरण देखाउने उदाहरणहरू
- **उपकरण कार्यान्वयन ढाँचाहरू**: विभिन्न जटिलताका साथ उपकरणहरू कार्यान्वयन गर्ने विभिन्न ढाँचाहरू

MCP C# SDK पूर्वावलोकन अवस्थामा छ र API हरू परिवर्तन हुन सक्छन्। हामी SDK विकासक्रमसँगै यस ब्लगलाई निरन्तर अपडेट गर्नेछौं।

### प्रमुख सुविधाहरू 
- [C# MCP Nuget ModelContextProtocol](https://www.nuget.org/packages/ModelContextProtocol)

- तपाईंको [पहिलो MCP सर्भर निर्माण गर्ने](https://devblogs.microsoft.com/dotnet/build-a-model-context-protocol-mcp-server-in-csharp/) तरिका।

पूर्ण C# कार्यान्वयन नमूनाहरूका लागि, [आधिकारिक C# SDK नमुना रिपोजिटरी](https://github.com/modelcontextprotocol/csharp-sdk) भ्रमण गर्नुहोस्।

## API व्यवस्थापन

Azure API Management एक उत्कृष्ट समाधान हो जसले MCP सर्भरहरूलाई सुरक्षित कसरी गर्ने भन्ने प्रश्नको जवाफ दिन्छ। यसको विचार हो कि तपाईंको MCP सर्भरको अगाडि Azure API Management इन्स्ट्यान्स राखेर तपाईंलाई आवश्यक पर्ने सुविधाहरू जस्तै:

- दर सीमा निर्धारण
- टोकन व्यवस्थापन
- निगरानी
- लोड ब्यालेन्सिङ
- सुरक्षा

यहाँ Azure नमुना छ जसले ठीक त्यही गर्छ, अर्थात् [MCP सर्भर सिर्जना गरी Azure API Management द्वारा सुरक्षित गर्ने](https://github.com/Azure-Samples/remote-mcp-apim-functions-python)।

## Azure मा Remote MCP Server तैनाथ गर्ने

अहिले हामीले पहिले उल्लेख गरेको नमुना तैनाथ गर्न सकिन्छ कि सकिंदैन हेर्ने प्रयास गरौं:

1. रिपोजिटरी क्लोन गर्नुहोस्

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

2. `Microsoft.App` पाउँदा दर्ता गर्नुहोस्:

    `` resource provider.
    * If you are using Azure CLI, run `az provider register --namespace Microsoft.App --wait`.
    * If you are using Azure PowerShell, run `Register-AzResourceProvider -ProviderNamespace Microsoft.App`. Then run `(Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState`

    केही समयपछि दर्ता पूरा भएको छ कि छैन जाँच्न।

3. यो [azd](https://aka.ms/azd) कमाण्ड चलाउनुहोस् जसले API व्यवस्थापन सेवा, function app (कोडसहित) र अन्य आवश्यक Azure स्रोतहरू प्रावधान गर्नेछ

    ```shell
    azd up
    ```

    यसले Azure मा सबै क्लाउड स्रोतहरू तैनाथ गर्नेछ।

### MCP Inspector सँग आफ्नो सर्भर परीक्षण गर्ने

1. **नयाँ टर्मिनल विन्डोमा**, MCP Inspector इन्स्टल र चलाउनुहोस्

    ```shell
    npx @modelcontextprotocol/inspector
    ```

    तपाईंले यस्तै इन्टरफेस देख्नु पर्नेछ:

    ![Connect to Node inspector](../../../translated_images/connect.141db0b2bd05f096fb1dd91273771fd8b2469d6507656c3b0c9df4b3c5473929.ne.png) 

2. एपले देखाएको URL (जस्तै http://127.0.0.1:6274/#resources) मा CTRL क्लिक गरेर MCP Inspector वेब एप लोड गर्नुहोस्
3. ट्रान्सपोर्ट प्रकारलाई `SSE`
1. Set the URL to your running API Management SSE endpoint displayed after `azd up` मा सेट गरेर **Connect** गर्नुहोस्:

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

5. **List Tools**। कुनै उपकरणमा क्लिक गरेर **Run Tool** गर्नुहोस्।

यदि सबै चरणहरू सफल भए, तपाईं MCP सर्भरसँग जडान हुनुहुन्छ र उपकरण कल गर्न सक्षम हुनुहुन्छ।

## Azure का लागि MCP सर्भरहरू

[Remote-mcp-functions](https://github.com/Azure-Samples/remote-mcp-functions-dotnet): यी रिपोजिटरीहरू Azure Functions प्रयोग गरेर Python, C# .NET वा Node/TypeScript मा अनुकूलित रिमोट MCP सर्भरहरू निर्माण र तैनाथ गर्नका लागि क्विकस्टार्ट टेम्प्लेटहरू हुन्।

नमूनाहरूले विकासकर्ताहरूलाई पूर्ण समाधान प्रदान गर्छन् जसले निम्न कार्यहरू गर्न सकिन्छ:

- स्थानीय रूपमा निर्माण र चलाउनुहोस्: स्थानीय मेसिनमा MCP सर्भर विकास र डिबग गर्नुहोस्
- Azure मा तैनाथ गर्नुहोस्: सजिलै क्लाउडमा azd up कमाण्डबाट तैनाथ गर्नुहोस्
- क्लाइन्टहरूबाट जडान गर्नुहोस्: विभिन्न क्लाइन्टहरूबाट MCP सर्भरसँग जडान हुनुहोस् जसमा VS Code को Copilot एजेन्ट मोड र MCP Inspector उपकरण समावेश छन्

### प्रमुख सुविधाहरू:

- डिजाइनबाटै सुरक्षा: MCP सर्भर कुञ्जीहरू र HTTPS प्रयोग गरेर सुरक्षित गरिएको छ
- प्रमाणीकरण विकल्पहरू: OAuth समर्थन गर्दछ, निर्मित प्रमाणीकरण र/वा API Management सँग
- नेटवर्क पृथकीकरण: Azure Virtual Networks (VNET) प्रयोग गरेर नेटवर्क पृथकीकरण अनुमति दिन्छ
- सर्भरलेस वास्तुकला: Azure Functions प्रयोग गरेर स्केलेबल, घटना-चालित कार्यान्वयन
- स्थानीय विकास: व्यापक स्थानीय विकास र डिबग समर्थन
- सरल तैनाथी: Azure मा सहज तैनाथी प्रक्रिया

रिपोजिटरीमा सबै आवश्यक कन्फिगरेसन फाइलहरू, स्रोत कोड, र पूर्वाधार परिभाषाहरू समावेश छन् जसले उत्पादन-तय MCP सर्भर कार्यान्वयन छिटो सुरु गर्न सहयोग पुर्‍याउँछ।

- [Azure Remote MCP Functions Python](https://github.com/Azure-Samples/remote-mcp-functions-python) - Azure Functions सँग Python प्रयोग गरेर MCP को नमुना कार्यान्वयन

- [Azure Remote MCP Functions .NET](https://github.com/Azure-Samples/remote-mcp-functions-dotnet) - Azure Functions सँग C# .NET प्रयोग गरेर MCP को नमुना कार्यान्वयन

- [Azure Remote MCP Functions Node/Typescript](https://github.com/Azure-Samples/remote-mcp-functions-typescript) - Azure Functions सँग Node/TypeScript प्रयोग गरेर MCP को नमुना कार्यान्वयन

## मुख्य सिकाइका बुँदाहरू

- MCP SDK हरूले भाषा-विशिष्ट उपकरणहरू प्रदान गर्छन् जसले बलियो MCP समाधानहरू कार्यान्वयन गर्न मद्दत गर्छ
- डिबग र परीक्षण प्रक्रिया विश्वसनीय MCP अनुप्रयोगहरूको लागि महत्वपूर्ण छ
- पुन: प्रयोग गर्न मिल्ने प्रम्प्ट टेम्प्लेटहरूले निरन्तर AI अन्तरक्रियालाई सक्षम पार्छ
- राम्रो डिजाइन गरिएका कार्यप्रवाहहरूले विभिन्न उपकरणहरू प्रयोग गरी जटिल कार्यहरू समन्वय गर्न सक्छन्
- MCP समाधान कार्यान्वयन गर्दा सुरक्षा, प्रदर्शन, र त्रुटि ह्यान्डलिङको विचार आवश्यक छ

## अभ्यास

आफ्नो क्षेत्रमा वास्तविक समस्यालाई सम्बोधन गर्ने व्यावहारिक MCP कार्यप्रवाह डिजाइन गर्नुहोस्:

1. यस समस्यालाई समाधान गर्न उपयोगी हुने ३-४ उपकरणहरू पहिचान गर्नुहोस्
2. यी उपकरणहरू कसरी अन्तरक्रिया गर्छन् देखाउने कार्यप्रवाह आरेख सिर्जना गर्नुहोस्
3. तपाईंको रोजाइको भाषामा एउटा उपकरणको आधारभूत संस्करण कार्यान्वयन गर्नुहोस्
4. मोडेलले तपाईंको उपकरण प्रभावकारी रूपमा प्रयोग गर्न मद्दत गर्ने प्रम्प्ट टेम्प्लेट सिर्जना गर्नुहोस्
