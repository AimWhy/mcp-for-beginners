# व्यावहारिक कार्यान्वयन

व्यावहारिक कार्यान्वयन वह स्थान है जहां मॉडल संदर्भ प्रोटोकॉल (MCP) की शक्ति ठोस रूप में प्रकट होती है। जबकि MCP के पीछे के सिद्धांत और वास्तुकला को समझना महत्वपूर्ण है, वास्तविक मूल्य तब उभरता है जब आप इन अवधारणाओं को लागू करके वास्तविक दुनिया की समस्याओं को हल करने वाले समाधान बनाते, परीक्षण करते और तैनात करते हैं। यह अध्याय अवधारणात्मक ज्ञान और व्यावहारिक विकास के बीच की खाई को पाटता है, आपको MCP-आधारित अनुप्रयोगों को जीवन में लाने की प्रक्रिया के माध्यम से मार्गदर्शन करता है।

चाहे आप बुद्धिमान सहायकों का विकास कर रहे हों, व्यवसाय वर्कफ़्लो में AI का एकीकरण कर रहे हों, या डेटा प्रोसेसिंग के लिए कस्टम टूल बना रहे हों, MCP एक लचीला आधार प्रदान करता है। इसकी भाषा-अज्ञेय डिजाइन और लोकप्रिय प्रोग्रामिंग भाषाओं के लिए आधिकारिक SDK इसे विभिन्न प्रकार के डेवलपर्स के लिए सुलभ बनाते हैं। इन SDK का उपयोग करके, आप विभिन्न प्लेटफार्मों और वातावरणों में अपने समाधानों को जल्दी से प्रोटोटाइप, पुनरावृत्त और स्केल कर सकते हैं।

आगामी अनुभागों में, आपको C#, Java, TypeScript, JavaScript, और Python में MCP को लागू करने के व्यावहारिक उदाहरण, नमूना कोड, और तैनाती रणनीतियाँ मिलेंगी। आप यह भी सीखेंगे कि अपने MCP सर्वरों को कैसे डिबग और परीक्षण करें, API का प्रबंधन करें, और Azure का उपयोग करके समाधान को क्लाउड में तैनात करें। ये व्यावहारिक संसाधन आपके सीखने को तेज करने और आपको आत्मविश्वास से मजबूत, उत्पादन-तैयार MCP अनुप्रयोगों को बनाने में मदद करने के लिए डिज़ाइन किए गए हैं।

## अवलोकन

यह पाठ कई प्रोग्रामिंग भाषाओं में MCP कार्यान्वयन के व्यावहारिक पहलुओं पर केंद्रित है। हम C#, Java, TypeScript, JavaScript, और Python में MCP SDK का उपयोग करके मजबूत अनुप्रयोग बनाने, MCP सर्वरों को डिबग और परीक्षण करने, और पुन: प्रयोज्य संसाधन, प्रॉम्प्ट और उपकरण बनाने की खोज करेंगे।

## सीखने के उद्देश्य

इस पाठ के अंत तक, आप सक्षम होंगे:
- विभिन्न प्रोग्रामिंग भाषाओं में आधिकारिक SDK का उपयोग करके MCP समाधान लागू करना
- व्यवस्थित रूप से MCP सर्वरों को डिबग और परीक्षण करना
- सर्वर सुविधाओं (संसाधन, प्रॉम्प्ट और उपकरण) बनाना और उपयोग करना
- जटिल कार्यों के लिए प्रभावी MCP वर्कफ़्लो डिज़ाइन करना
- प्रदर्शन और विश्वसनीयता के लिए MCP कार्यान्वयन को अनुकूलित करना

## आधिकारिक SDK संसाधन

मॉडल संदर्भ प्रोटोकॉल कई भाषाओं के लिए आधिकारिक SDK प्रदान करता है:

- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk)
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) 
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk)
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk)
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk)

## MCP SDK के साथ काम करना

यह अनुभाग कई प्रोग्रामिंग भाषाओं में MCP को लागू करने के व्यावहारिक उदाहरण प्रदान करता है। आप भाषा के अनुसार व्यवस्थित `samples` निर्देशिका में नमूना कोड पा सकते हैं।

### उपलब्ध नमूने

भंडार में निम्नलिखित भाषाओं में नमूना कार्यान्वयन शामिल हैं:

- C#
- Java
- TypeScript
- JavaScript
- Python

प्रत्येक नमूना उस विशिष्ट भाषा और पारिस्थितिकी तंत्र के लिए प्रमुख MCP अवधारणाओं और कार्यान्वयन पैटर्न को प्रदर्शित करता है।

## कोर सर्वर सुविधाएँ

MCP सर्वर इन सुविधाओं के किसी भी संयोजन को लागू कर सकते हैं:

### संसाधन
संसाधन उपयोगकर्ता या AI मॉडल के लिए संदर्भ और डेटा प्रदान करते हैं:
- दस्तावेज़ भंडार
- ज्ञान आधार
- संरचित डेटा स्रोत
- फ़ाइल सिस्टम

### प्रॉम्प्ट
प्रॉम्प्ट उपयोगकर्ताओं के लिए टेम्पलेटेड संदेश और वर्कफ़्लो होते हैं:
- पूर्व-निर्धारित वार्तालाप टेम्पलेट
- निर्देशित बातचीत पैटर्न
- विशेष संवाद संरचनाएँ

### उपकरण
उपकरण AI मॉडल के लिए निष्पादन के कार्य होते हैं:
- डेटा प्रोसेसिंग उपयोगिताएँ
- बाहरी API एकीकरण
- गणनात्मक क्षमताएँ
- खोज कार्यक्षमता

## नमूना कार्यान्वयन: C#

आधिकारिक C# SDK भंडार में MCP के विभिन्न पहलुओं को प्रदर्शित करने वाले कई नमूना कार्यान्वयन शामिल हैं:

- **बेसिक MCP क्लाइंट**: एक साधारण उदाहरण जो दिखाता है कि MCP क्लाइंट कैसे बनाया जाता है और उपकरणों को कैसे कॉल किया जाता है
- **बेसिक MCP सर्वर**: बुनियादी उपकरण पंजीकरण के साथ न्यूनतम सर्वर कार्यान्वयन
- **उन्नत MCP सर्वर**: उपकरण पंजीकरण, प्रमाणीकरण, और त्रुटि हैंडलिंग के साथ पूर्ण-विशेषता सर्वर
- **ASP.NET एकीकरण**: ASP.NET Core के साथ एकीकरण का प्रदर्शन करने वाले उदाहरण
- **उपकरण कार्यान्वयन पैटर्न**: विभिन्न जटिलता स्तरों के साथ उपकरणों को लागू करने के विभिन्न पैटर्न

MCP C# SDK पूर्वावलोकन में है और API बदल सकते हैं। जैसे-जैसे SDK विकसित होता रहेगा, हम इस ब्लॉग को लगातार अपडेट करते रहेंगे।

### प्रमुख विशेषताएँ 
- [C# MCP Nuget ModelContextProtocol](https://www.nuget.org/packages/ModelContextProtocol)

- अपना [पहला MCP सर्वर बनाना](https://devblogs.microsoft.com/dotnet/build-a-model-context-protocol-mcp-server-in-csharp/)।

## दूरस्थ MCP सर्वर को Azure पर तैनात करें

आइए देखें कि क्या हम पहले उल्लेखित नमूने को तैनात कर सकते हैं:

1. रेपो क्लोन करें

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

1. `Microsoft.App` resource provider.
    * If you are using Azure CLI, run `az provider register --namespace Microsoft.App --wait`.
    * If you are using Azure PowerShell, run `Register-AzResourceProvider -ProviderNamespace Microsoft.App`. Then run `(Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState` को कुछ समय बाद पंजीकरण पूरा होने की जांच करने के लिए रजिस्टर करें।

2. API प्रबंधन सेवा, फ़ंक्शन ऐप (कोड के साथ) और सभी अन्य आवश्यक Azure संसाधनों को प्रदान करने के लिए इस [azd](https://aka.ms/azd) कमांड को चलाएँ

    ```shell
    azd up
    ```

    यह कमांड Azure पर सभी क्लाउड संसाधनों को तैनात करना चाहिए

### MCP इंस्पेक्टर के साथ अपने सर्वर का परीक्षण करना

1. एक **नए टर्मिनल विंडो** में, MCP इंस्पेक्टर इंस्टॉल और चलाएँ

    ```shell
    npx @modelcontextprotocol/inspector
    ```

    आपको एक इंटरफ़ेस देखना चाहिए जो इस तरह दिखता है:

    ![Connect to Node inspector](../../../translated_images/connect.126df3a6deef0d2e0850b1a5cf0547c8fc4e2e9e0b354238d0a9dbe7893726fa.hi.png)

1. ऐप द्वारा प्रदर्शित URL से MCP इंस्पेक्टर वेब ऐप लोड करने के लिए CTRL क्लिक करें (जैसे http://127.0.0.1:6274/#resources)
1. परिवहन प्रकार को `SSE`
1. Set the URL to your running API Management SSE endpoint displayed after `azd up` पर सेट करें और **कनेक्ट** करें:

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

5. **उपकरण सूचीबद्ध करें**। एक उपकरण पर क्लिक करें और **उपकरण चलाएँ**।

यदि सभी चरण काम कर चुके हैं, तो अब आप MCP सर्वर से जुड़े हुए हैं और आप एक उपकरण को कॉल कर चुके हैं।

## Azure के लिए MCP सर्वर 

[Remote-mcp-functions](https://github.com/Azure-Samples/remote-mcp-functions-dotnet): यह सेट रेपो कस्टम दूरस्थ MCP (मॉडल संदर्भ प्रोटोकॉल) सर्वर बनाने और तैनात करने के लिए क्विकस्टार्ट टेम्पलेट हैं, जो Azure Functions का उपयोग करते हुए Python, C# .NET या Node/TypeScript के साथ हैं।

नमूने डेवलपर्स को निम्नलिखित की अनुमति देने वाला एक पूर्ण समाधान प्रदान करते हैं:

- स्थानीय रूप से निर्माण और चलाना: स्थानीय मशीन पर MCP सर्वर विकसित और डिबग करें
- Azure पर तैनात करें: सरल azd up कमांड के साथ आसानी से क्लाउड पर तैनात करें
- क्लाइंट्स से कनेक्ट करें: VS Code के Copilot एजेंट मोड और MCP इंस्पेक्टर टूल सहित विभिन्न क्लाइंट्स से MCP सर्वर से कनेक्ट करें

### प्रमुख विशेषताएँ:

- डिज़ाइन द्वारा सुरक्षा: MCP सर्वर को कुंजियों और HTTPS का उपयोग करके सुरक्षित किया जाता है
- प्रमाणीकरण विकल्प: अंतर्निहित प्रमाणीकरण और/या API प्रबंधन का उपयोग करके OAuth का समर्थन करता है
- नेटवर्क पृथक्करण: Azure वर्चुअल नेटवर्क (VNET) का उपयोग करके नेटवर्क पृथक्करण की अनुमति देता है
- सर्वर रहित वास्तुकला: स्केलेबल, इवेंट-चालित निष्पादन के लिए Azure Functions का लाभ उठाता है
- स्थानीय विकास: व्यापक स्थानीय विकास और डिबगिंग समर्थन
- सरल तैनाती: Azure पर तैनाती प्रक्रिया को सुव्यवस्थित करता है

भंडार में सभी आवश्यक कॉन्फ़िगरेशन फ़ाइलें, स्रोत कोड, और इन्फ्रास्ट्रक्चर परिभाषाएँ शामिल हैं ताकि उत्पादन-तैयार MCP सर्वर कार्यान्वयन के साथ जल्दी से शुरुआत की जा सके।

- [Azure Remote MCP Functions Python](https://github.com/Azure-Samples/remote-mcp-functions-python) - Python के साथ Azure Functions का उपयोग करके MCP का नमूना कार्यान्वयन

- [Azure Remote MCP Functions .NET](https://github.com/Azure-Samples/remote-mcp-functions-dotnet) - C# .NET के साथ Azure Functions का उपयोग करके MCP का नमूना कार्यान्वयन

- [Azure Remote MCP Functions Node/Typescript](https://github.com/Azure-Samples/remote-mcp-functions-typescript) - Node/TypeScript के साथ Azure Functions का उपयोग करके MCP का नमूना कार्यान्वयन।

## प्रमुख निष्कर्ष

- MCP SDK मजबूत MCP समाधान को लागू करने के लिए भाषा-विशिष्ट उपकरण प्रदान करते हैं
- विश्वसनीय MCP अनुप्रयोगों के लिए डिबगिंग और परीक्षण प्रक्रिया महत्वपूर्ण है
- पुन: प्रयोज्य प्रॉम्प्ट टेम्पलेट्स सुसंगत AI इंटरैक्शन को सक्षम करते हैं
- अच्छी तरह से डिज़ाइन किए गए वर्कफ़्लो कई उपकरणों का उपयोग करके जटिल कार्यों को व्यवस्थित कर सकते हैं
- MCP समाधान लागू करने के लिए सुरक्षा, प्रदर्शन, और त्रुटि हैंडलिंग पर विचार करने की आवश्यकता होती है

## अभ्यास

अपने डोमेन में एक वास्तविक समस्या का समाधान करने वाला एक व्यावहारिक MCP वर्कफ़्लो डिज़ाइन करें:

1. उन 3-4 उपकरणों की पहचान करें जो इस समस्या को हल करने में उपयोगी होंगे
2. इन उपकरणों के बीच बातचीत दिखाने वाला एक वर्कफ़्लो आरेख बनाएं
3. अपनी पसंदीदा भाषा का उपयोग करके उपकरणों में से एक का एक बुनियादी संस्करण लागू करें
4. एक प्रॉम्प्ट टेम्पलेट बनाएं जो आपके उपकरण का प्रभावी ढंग से उपयोग करने में मॉडल की मदद करेगा
