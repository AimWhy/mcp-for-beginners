# יצירת לקוח עם LLM

עד כה, ראיתם כיצד ליצור שרת ולקוח. הלקוח היה יכול לקרוא לשרת במפורש כדי לרשום את הכלים, המשאבים וההנחיות שלו. עם זאת, זו לא גישה מאוד מעשית. המשתמש שלך חי בעידן הסוכני ומצפה להשתמש בהנחיות ולתקשר עם LLM כדי לעשות זאת. עבור המשתמש שלך, לא משנה אם אתה משתמש ב-MCP או לא כדי לאחסן את היכולות שלך, אבל הם כן מצפים להשתמש בשפה טבעית כדי לתקשר. אז איך פותרים את זה? הפתרון הוא להוסיף LLM ללקוח.

## סקירה כללית

בשיעור זה נתמקד בהוספת LLM ללקוח שלך ונראה כיצד זה מספק חווית משתמש טובה בהרבה.

## מטרות למידה

בסוף השיעור הזה, תוכל:

- ליצור לקוח עם LLM.
- לתקשר בצורה חלקה עם שרת MCP באמצעות LLM.
- לספק חווית משתמש טובה יותר בצד הלקוח.

## גישה

בואו ננסה להבין את הגישה שעלינו לנקוט. הוספת LLM נשמעת פשוטה, אבל האם אנחנו באמת נעשה את זה?

כך הלקוח יתקשר עם השרת:

1. הקמת חיבור עם השרת.

1. רישום יכולות, הנחיות, משאבים וכלים, ושמירת הסכמות שלהם.

1. הוספת LLM והעברת היכולות השמורות והסכמות שלהם בפורמט שה-LLM מבין.

1. טיפול בהנחיית משתמש על ידי העברתה ל-LLM יחד עם הכלים שרשומים על ידי הלקוח.

מצוין, עכשיו אנחנו מבינים איך אפשר לעשות זאת ברמה גבוהה, בואו ננסה זאת בתרגיל הבא.

## משימה

קח את הקוד מהתרגיל ובנה את השרת עם עוד כלים. לאחר מכן צור לקוח עם LLM, כמו בתרגיל, ובדוק אותו עם הנחיות שונות כדי לוודא שכל הכלים של השרת נקראים באופן דינמי. דרך זו של בניית לקוח משמעותה שלמשתמש הקצה תהיה חווית משתמש נהדרת כי הם יכולים להשתמש בהנחיות, במקום פקודות לקוח מדויקות, ולהיות בלתי מודעים לכל שרת MCP שנקרא.
